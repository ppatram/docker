# Video 8: Pods and YAML Fundamentals

📺 [Watch on YouTube](https://www.youtube.com/watch?v=_f9ql2Y5Xcc)

## Overview

This video covers creating Pods in Kubernetes using both imperative and declarative approaches. It introduces YAML fundamentals (data types, indentation, syntax), the four top-level fields of a Kubernetes manifest (apiVersion, kind, metadata, spec), and essential kubectl commands for managing pods including describe, exec, edit, dry-run, and output formatting.

## Terminal Sessions

### Creating a Pod - Imperative Way

```bash
# Create an nginx pod imperatively
kubectl run nginx-pod --image=nginx

# Verify the pod is running
kubectl get pods
```

### YAML Fundamentals

```yaml
# This is a comment
# YAML supports strings, integers, lists, dictionaries

employee:
  name: Piyush
  age: 34
  address: 107 XYZ

# List example (using dash)
employee:
  - name: Piyush
    age: 34
  - name: AnotherName
    age: 1.8

# Nested lists
employee:
  - name: Piyush
    address:
      - old address
      - new address
```

### Pod YAML - Declarative Way

```yaml
# pod.yaml - A sample pod YAML
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  labels:
    env: demo
    type: frontend
spec:
  containers:
    - name: nginx-container
      image: nginx
      ports:
        - containerPort: 80
```

### Applying the Pod YAML

```bash
# Check API version for pod
kubectl explain pod

# Rename file
mv seven-yaml.yaml pod.yaml

# Create pod from YAML file
kubectl create -f pod.yaml

# If pod already exists, delete it first
kubectl delete pod nginx-pod

# Create again
kubectl create -f pod.yaml

# Verify
kubectl get pods
```

### Updating and Troubleshooting Pods

```bash
# Apply changes to a running pod (update YAML and apply)
kubectl apply -f pod.yaml

# Check pod status (shows errors like ImagePullBackOff)
kubectl get pods

# Describe pod to see detailed events and errors
kubectl describe pod nginx-pod

# Edit pod directly on the cluster (opens vi editor)
kubectl edit pod nginx-pod
# Make changes in vi, then save with :wq!

# Verify pod is running again
kubectl get pods
```

### Exec into a Pod

```bash
# Open an interactive shell inside the pod
kubectl exec -it nginx-pod -- sh

# Inside the container:
pwd
ls
exit
```

### Generating YAML from Imperative Commands (Dry Run)

```bash
# Dry run - shows what would happen without applying
kubectl run nginx --image=nginx --dry-run=client

# Output as YAML
kubectl run nginx --image=nginx --dry-run=client -o yaml

# Redirect to a file
kubectl run nginx --image=nginx --dry-run=client -o yaml > pod-new.yaml

# Output as JSON
kubectl run nginx --image=nginx --dry-run=client -o json > pod-new.json
```

### Additional Useful Commands

```bash
# Describe a pod (full details including node, conditions, events)
kubectl describe pod nginx-pod

# Show labels on pods
kubectl get pods nginx-pod --show-labels

# Wide output (shows IP, node, and other details)
kubectl get pods -o wide

# Wide output for nodes (shows OS, kernel, container runtime, IPs)
kubectl get nodes -o wide

# Delete a pod
kubectl delete pod nginx-pod
```

## Key Points

- **Imperative**: `kubectl run` - quick commands for creating resources
- **Declarative**: Create YAML files and use `kubectl create -f` or `kubectl apply -f`
- Four top-level fields: `apiVersion`, `kind`, `metadata`, `spec`
- YAML is case-sensitive
- Use `kubectl explain <resource>` to check API version and fields
- Use `--dry-run=client -o yaml` to generate YAML templates
