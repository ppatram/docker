# Video 20 of 59 - ConfigMaps and Secrets in Kubernetes

📺 https://www.youtube.com/watch?v=Q9fHJLSyd7Q

## Overview

This video covers ConfigMaps and Secrets in Kubernetes — how to externalize configuration from pod definitions, create ConfigMaps imperatively and declaratively, and inject them into pods as environment variables. It demonstrates multiple injection methods: single key reference, envFrom for all keys, and briefly mentions volume mounts (covered in a later storage video).

## Terminal Sessions

### Setup and basic environment variable in pod

```bash
# Navigate to day 19 folder
cd day19/

# Apply pod with inline environment variable
kubectl apply -f pod.yaml

# Check pod status
kubectl get pods

# Exec into the pod and verify environment variable
kubectl exec -it <pod-name> -- sh
echo $FIRST_NAME
# Prints the value

```

### Create ConfigMap imperatively (from-literal)

```bash
# Create a ConfigMap with key-value pairs
kubectl create cm app-cm \
  --from-literal=first_name=Piyush \
  --from-literal=last_name=Sachdeva

# Verify ConfigMap was created
kubectl get cm

# Describe ConfigMap to see data
kubectl describe cm/app-cm
# Shows: first_name: Piyush, last_name: Sachdeva
```

### Inject ConfigMap into pod (single key reference)

```bash
# Apply updated pod YAML with configMapKeyRef
kubectl apply -f pod.yaml

# Force delete and recreate if needed
kubectl delete pod <pod-name>
kubectl apply -f pod.yaml

# Verify pods are running
kubectl get pods

# Exec into pod and check injected value
kubectl exec -it <pod-name> -- sh
echo $FIRST_NAME
# Prints the value from config map
exit

# Describe pod to see environment variable source
kubectl describe pod <pod-name>
# Shows: FIRST_NAME set to the key first_name of config map app-cm
```

### Create ConfigMap declaratively (dry-run to generate YAML)

```bash
# Generate ConfigMap YAML using dry-run
kubectl create cm app-cm \
  --from-literal=first_name=Piyush \
  --from-literal=last_name=Sachdeva \
  --dry-run=client -o yaml > cm.yaml

# View the generated YAML
cat cm.yaml
```

### Create ConfigMap from file

```bash
# Create ConfigMap from a file (alternative to from-literal for many keys)
kubectl create cm app-cm --from-file=app.config
```

### Update ConfigMap (force replace)

```bash
# Force update a ConfigMap (cannot edit in place easily)
kubectl apply -f cm.yaml --force
```

### Cleanup

```bash
# Delete previous pods
kubectl delete pod <pod-name>
```

## YAML Manifests

### Pod with inline environment variable

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: busybox
      command: ["sh", "-c", "echo The app is running && sleep 3600"]
      env:
        - name: FIRST_NAME
          value: "Piyush"
```

### Pod with ConfigMap key reference (single key)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: busybox
      command: ["sh", "-c", "echo The app is running && sleep 3600"]
      env:
        - name: FIRST_NAME
          valueFrom:
            configMapKeyRef:
              name: app-cm
              key: first_name
```

### Pod with envFrom (all keys from ConfigMap)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: busybox
      command: ["sh", "-c", "echo The app is running && sleep 3600"]
      envFrom:
        - configMapRef:
            name: app-cm
```

### ConfigMap declarative YAML

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-cm
data:
  first_name: Piyush
  last_name: Sachdeva
```

### Pod with ConfigMap mounted as volume (preview)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
spec:
  containers:
    - name: myapp
      image: busybox
      command: ["sh", "-c", "echo The app is running && sleep 3600"]
      volumeMounts:
        - name: config-volume
          mountPath: /etc/config
  volumes:
    - name: config-volume
      configMap:
        name: app-cm
```

## Key Concepts

- **ConfigMap**: Kubernetes object to store non-sensitive configuration data as key-value pairs, externalized from pod definitions.
- **Imperative creation**: `kubectl create cm <name> --from-literal=key=value`
- **From file**: `kubectl create cm <name> --from-file=<filename>` — useful when there are many key-value pairs.
- **Injection methods**:
  1. `valueFrom.configMapKeyRef` — inject a single key as an environment variable
  2. `envFrom.configMapRef` — inject ALL keys from a ConfigMap as environment variables
  3. Volume mount — mount ConfigMap as a file/directory in the container (covered in storage section)
- **Secrets**: Similar to ConfigMaps but for sensitive data. Can be mounted as volumes (best practice) or injected as env vars. Covered in more detail in storage section.
- **Updating ConfigMaps**: Use `--force` flag or delete and recreate. Pod restart may be needed to pick up changes.
