# Video 43 of 59 - Hosting a Private Docker Container Registry on Kubernetes

📺 **YouTube**: [https://www.youtube.com/watch?v=Py-vsaD0WT4](https://www.youtube.com/watch?v=Py-vsaD0WT4)

## Overview

This video is a real-time project that demonstrates hosting a private Docker container registry locally on Kubernetes using the CNCF Distribution project (registry:2). It covers generating self-signed TLS certificates, creating authentication credentials, configuring Kubernetes Secrets, Persistent Volumes, Deployments, and Services. The project implements best practices including TLS encryption, basic authentication, persistent storage, and image pull secrets.

## Terminal Sessions

### Initial cluster check

```bash
kubectl get nodes
```

### Create directory structure for the registry project

```bash
mkdir registry
cd registry
mkdir certs
mkdir auth
```

### Generate self-signed TLS certificates

```bash
openssl req -x509 -newkey rsa:4096 -days 365 -nodes \
  -keyout certs/tls.key -out certs/tls.crt \
  -subj "/CN=my-registry" \
  -addext "subjectAltName=DNS:my-registry"
```

### Generate htpasswd authentication file

```bash
docker run --entrypoint htpasswd httpd:2 -Bbn myuser mypasswd > auth/htpasswd

# Verify the auth file
cat auth/htpasswd
```

### Create TLS secret in Kubernetes

```bash
kubectl create secret tls certs-secret \
  --cert=/home/ubuntu/demo-reg/registry/certs/tls.crt \
  --key=/home/ubuntu/demo-reg/registry/certs/tls.key

# Verify
kubectl get secrets
kubectl describe secret certs-secret
```

### Create generic secret for authentication

```bash
kubectl create secret generic auth-secret \
  --from-file=/home/ubuntu/demo-reg/registry/auth/htpasswd

# Verify
kubectl get secrets
```

### Create Persistent Volume and PVC

```bash
# Create host path directory
mkdir -p /home/ubuntu/repos

# Apply volume configuration
kubectl apply -f volume.yaml

# Verify
kubectl get pv
kubectl get pvc
```

### Apply Deployment

```bash
kubectl apply -f deploy.yaml
kubectl get pods
```

### Apply Service

```bash
kubectl apply -f service.yaml
kubectl get svc
kubectl describe svc docker-registry
```

### Configure DNS and environment variables

```bash
# Export registry details
export REGISTRY_NAME=my-registry
export REGISTRY_IP=<cluster-ip>

# Add DNS entry on all nodes (control plane + workers)
sudo vi /etc/hosts
# Add: <REGISTRY_IP>  my-registry
```

### Configure nodes to trust the self-signed certificate

```bash
# On ALL nodes (control plane and workers):

# Copy TLS certificate to system trust store
sudo cp tls.crt /usr/local/share/ca-certificates/tls.crt
sudo update-ca-certificates

# If issues, do fresh update:
sudo rm /usr/local/share/ca-certificates/tls.crt
sudo update-ca-certificates --fresh
sudo vi /usr/local/share/ca-certificates/tls.crt
# Paste certificate content
sudo update-ca-certificates
```

### Configure Docker to trust the self-signed certificate

```bash
# On ALL nodes, create Docker cert directory:
sudo mkdir -p /etc/docker/certs.d/my-registry:5000

# Copy cert to Docker trust directory:
sudo cp tls.crt /etc/docker/certs.d/my-registry:5000/tls.crt
```

### Test Docker login

```bash
docker login my-registry:5000
# Username: myuser
# Password: mypasswd
# Expected: Login Succeeded
```

### Push an image to the private registry

```bash
# Pull a sample image
docker pull nginx

# Tag it for the private registry
docker tag nginx:latest my-registry:5000/my-nginx:v2

# Push to the private registry
docker push my-registry:5000/my-nginx:v2
```

### Verify image in registry pod

```bash
# Exec into registry pod
kubectl exec -it <registry-pod-name> -- sh

# Navigate to repository storage
ls /var/lib/registry/docker/registry/v2/repositories/

# Exit pod
exit
```

### Create Docker registry secret for image pull

```bash
kubectl create secret docker-registry nginx-secret \
  --docker-server=my-registry:5000 \
  --docker-username=myuser \
  --docker-password=mypasswd
```

### Deploy a pod using the private registry image

```bash
# Apply pod manifest (with imagePullSecrets)
kubectl apply -f new.yaml

# Verify
kubectl get pods
kubectl get pods -o wide

# Test the nginx pod
curl <pod-ip>
```

## YAML Files

### volume.yaml - Persistent Volume and PVC

```yaml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: registry-pv
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: /home/ubuntu/repos
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: registry-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
```

### deploy.yaml - Registry Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: registry
spec:
  replicas: 2
  selector:
    matchLabels:
      app: registry
  template:
    metadata:
      labels:
        app: registry
    spec:
      containers:
        - name: registry
          image: registry:2.8.2
          ports:
            - containerPort: 5000
          volumeMounts:
            - name: repo-volume
              mountPath: /var/lib/registry
            - name: certs-volume
              mountPath: /certs
            - name: auth-volume
              mountPath: /auth
          env:
            - name: REGISTRY_HTTP_TLS_CERTIFICATE
              value: /certs/tls.crt
            - name: REGISTRY_HTTP_TLS_KEY
              value: /certs/tls.key
            - name: REGISTRY_AUTH
              value: htpasswd
            - name: REGISTRY_AUTH_HTPASSWD_REALM
              value: Registry Realm
            - name: REGISTRY_AUTH_HTPASSWD_PATH
              value: /auth/htpasswd
      volumes:
        - name: repo-volume
          persistentVolumeClaim:
            claimName: registry-pvc
        - name: certs-volume
          secret:
            secretName: certs-secret
        - name: auth-volume
          secret:
            secretName: auth-secret
```

### service.yaml - Registry Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: docker-registry
spec:
  selector:
    app: registry
  ports:
    - port: 5000
      targetPort: 5000
      nodePort: 30500
  type: NodePort
```

### new.yaml - Pod using private registry image

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
    - name: nginx
      image: my-registry:5000/my-nginx:v2
  imagePullSecrets:
    - name: nginx-secret
```
