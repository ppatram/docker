# Video 26 of 59 - Kubernetes Service Accounts

📺 https://www.youtube.com/watch?v=k2iCq7IlMKM

## Overview

This video covers Kubernetes Service Accounts — what they are, how to create them, and how they differ from human users. Service accounts are used by applications, bots, and automation tools (e.g., Jenkins, Prometheus, DataDog) to interact with the cluster programmatically. The video demonstrates creating service accounts, generating long-lived API tokens, assigning RBAC roles, and inspecting service account tokens mounted inside pods.

## Terminal Sessions

### Viewing Default Service Accounts

```bash
# List service accounts in the default namespace
kubectl get sa

# List default service accounts across all namespaces
kubectl get sa -A | grep default

# Describe the default service account
kubectl describe sa default

# View the default service account in YAML format
kubectl get sa default -o yaml
```

### Creating a Service Account

```bash
# Create a new service account for a build server (e.g., Jenkins)
kubectl create serviceaccount build-sa

# Describe the new service account
kubectl describe sa build-sa
```

### Creating a Long-Lived API Token (Secret)

```bash
# Create the secret YAML file
vi secret.yaml

# Apply the secret
kubectl apply -f secret.yaml

# List secrets
kubectl get secret

# Describe the secret to see the token
kubectl describe secret build-robot-secret
```

### Secret YAML for Service Account Token

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: build-robot-secret
  annotations:
    kubernetes.io/service-account.name: build-sa
type: kubernetes.io/service-account-token
```

### Testing Service Account Permissions

```bash
# Check if build-sa can list pods (before RBAC)
kubectl get pods --as build-sa
# Output: Error - user build-sa cannot list pods

# Check authorization
kubectl auth can-i get pods --as build-sa
# Output: no

# Create a role with list, get, watch verbs on pods
kubectl create role build-role --verb=list,get,watch --resource=pod

# Create a role binding to associate the role with the service account
kubectl create rolebinding rb --role=build-role --user=build-sa

# Verify permissions again
kubectl auth can-i get pods --as build-sa
# Output: yes

# List pods as the service account
kubectl get pods --as build-sa
```

### Inspecting Service Account Token Inside a Pod

```bash
# Describe a pod to see service account mount details
kubectl describe pod <pod-name>
# Look for: Service Account: default
# Look for: Mount: /var/run/secrets/kubernetes.io/serviceaccount

# Exec into the pod
kubectl exec -it <pod-name> -- bash

# Navigate to the mounted service account directory
cd /var/run/secrets/kubernetes.io/serviceaccount

# List files (token, namespace, ca.crt)
ls

# View the token
cat token
```

### Creating an Image Pull Secret

```bash
# Create a Docker registry secret
kubectl create secret docker-registry <secret-name> \
  --docker-server=<your-registry-server> \
  --docker-username=<your-username> \
  --docker-password=<your-password> \
  --docker-email=<your-email>
```

### Deleting a Service Account

```bash
# List service accounts
kubectl get sa

# Delete a service account
kubectl delete sa build-sa
```

### Pod YAML with Image Pull Secret

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-pod
spec:
  containers:
    - name: my-container
      image: my-private-image
  imagePullSecrets:
    - name: <secret-name>
```
