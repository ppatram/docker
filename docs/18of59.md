# Video 18 of 59 - Kubernetes Autoscaling (HPA, VPA, Cluster Autoscaler)

📺 https://www.youtube.com/watch?v=afUL5jGoLx0

## Overview

This video covers Kubernetes Autoscaling concepts including Horizontal Pod Autoscaler (HPA), Vertical Pod Autoscaler (VPA), Cluster Autoscaler, and event-based autoscaling (KEDA). It includes a hands-on demonstration of HPA by deploying a sample application, generating load, and observing automatic pod scaling based on CPU utilization. Note: Autoscaling is not on the CKA exam but is important for real-world Kubernetes usage.

## Terminal Sessions

### Verify cluster version and metrics server

```bash
# Check cluster version (upgraded to 1.30)
kubectl get nodes

# Verify metrics server is running (prerequisite for HPA)
kubectl get pods -n kube-system
```

### Deploy application and service for HPA testing

```bash
# Apply deployment and service YAML
kubectl apply -f deploy.yaml

# Check pods (should have 1 replica)
kubectl get pods

# Check service
kubectl get service
# Shows php-apache service on port 80
```

### Create HPA (imperative command)

```bash
# Delete existing HPA if it exists
kubectl delete hpa php-apache

# Create HPA targeting 50% CPU utilization, min 1 replica, max 10 replicas
kubectl autoscale deployment php-apache --cpu-percent=50 --min=1 --max=10

# Check HPA status
kubectl get hpa

# Watch HPA (shows CPU utilization vs target)
kubectl get hpa --watch
```

### Generate load to trigger autoscaling

```bash
# In a separate terminal window, run a load generator pod
kubectl run -it load-generator --rm --image=busybox --restart=Never -- /bin/sh -c "while sleep 0.01; do wget -q -O- http://php-apache; done"
```

### Observe autoscaling in action

```bash
# Watch HPA scaling (in original terminal)
kubectl get hpa --watch
# CPU utilization will increase beyond 50%, replicas will increase

# Check pods to see new replicas
kubectl get pods
# Shows 5-7+ pods running

# Check HPA status
kubectl get hpa
# Shows CPU at 117%, 7 replicas
```

### Stop load and observe scale-down

```bash
# Ctrl+C in the load generator terminal to stop traffic

# Watch HPA scale down (takes a few minutes)
kubectl get hpa
# CPU utilization decreases, replicas eventually return to 1
```

## YAML Manifests

### Deployment and Service for HPA demo

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: php-apache
spec:
  selector:
    matchLabels:
      run: php-apache
  template:
    metadata:
      labels:
        run: php-apache
    spec:
      containers:
        - name: php-apache
          image: registry.k8s.io/hpa-example
          ports:
            - containerPort: 80
          resources:
            requests:
              cpu: 200m
            limits:
              cpu: 500m
---
apiVersion: v1
kind: Service
metadata:
  name: php-apache
  labels:
    run: php-apache
spec:
  ports:
    - port: 80
  selector:
    run: php-apache
```

## Key Concepts

- **Horizontal Pod Autoscaler (HPA)**: Adds/removes pod replicas based on CPU/memory metrics. Built into Kubernetes. Requires Metrics Server.
- **Vertical Pod Autoscaler (VPA)**: Resizes pod resource requests/limits. Requires pod restart. Separate GitHub project.
- **Cluster Autoscaler**: Adds/removes nodes to the cluster. Available with managed cloud services (EKS, AKS, GKE).
- **Event-based Autoscaling (KEDA)**: Scales based on custom events (error codes, queue depth, etc.). CNCF project.
- **HPA default check interval**: Every 15 seconds (configurable in controller-manager).
- **Scale-down behavior**: Takes several minutes to scale down to avoid flapping.
- **Multiple objects in one YAML**: Use `---` separator between resources.
