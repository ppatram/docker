# Video 47 of 59 — Gateway API: Deep Dive and Implementation in Kubernetes

https://www.youtube.com/watch?v=qLL5wf0ShZY

## Overview

This video covers the Kubernetes Gateway API — a newer, more powerful replacement for Ingress. Topics include: differences between Ingress and Gateway API (advanced routing, protocol support, role-oriented architecture), Gateway API components (GatewayClass, Gateway, HTTPRoute), installing CRDs, deploying the NGINX Gateway Fabric controller, creating Gateway and HTTPRoute resources, TLS configuration, and testing traffic routing.

## Terminal Sessions

### Verify cluster status

```bash
kubectl cluster-info
kubectl get nodes
```

### Install Gateway API CRDs

```bash
kubectl apply -k "https://github.com/nginx/nginx-gateway-fabric/config/crd/gateway-api/standard?ref=v1.5.1"
```

### Install NGINX Gateway Fabric controller

```bash
# Apply the NGINX gateway fabric deployment
kubectl apply -f https://raw.githubusercontent.com/nginx/nginx-gateway-fabric/v1.5.1/deploy/default/deploy.yaml
```

### Deploy a sample application

```bash
# Create deployment and service
kubectl create deployment web-app --image=nginx --replicas=2
kubectl expose deployment web-app --port=80
```

### Create GatewayClass

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: GatewayClass
metadata:
  name: nginx
spec:
  controllerName: gateway.nginx.org/nginx-gateway-controller
EOF

# Verify
kubectl get gc
# or
kubectl get gatewayclass
```

### Create Gateway resource

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: nginx-gateway
  namespace: default
spec:
  gatewayClassName: nginx
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      allowedRoutes:
        namespaces:
          from: All
    - name: https
      protocol: HTTPS
      port: 443
      hostname: gateway.web.local
      tls:
        mode: Terminate
        certificateRefs:
          - kind: Secret
            name: web-tls-secret
            namespace: web-app
      allowedRoutes:
        namespaces:
          from: All
EOF

# Verify
kubectl get gateway
```

### Create HTTPRoute

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: web-route
  namespace: default
spec:
  parentRefs:
    - name: nginx-gateway
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: web-app
          port: 80
EOF

# Verify
kubectl get httproute
```

### Patch service to use NodePort

```bash
kubectl patch svc nginx-gateway-nginx-gateway-fabric -n nginx-gateway -p '{"spec":{"type":"NodePort","ports":[{"port":80,"nodePort":30080},{"port":443,"nodePort":30081}]}}'

# Verify service
kubectl get svc -n nginx-gateway
```

### Test the Gateway

```bash
# Get node IP
kubectl get nodes -o wide

# Test HTTP
curl http://gateway.web.local:30080

# Test HTTPS
curl -k https://gateway.web.local:30081

# Verbose output to confirm source
curl -kv https://gateway.web.local:30081
```

### Describe and troubleshoot

```bash
kubectl describe gateway nginx-gateway
kubectl describe httproute web-route
```
