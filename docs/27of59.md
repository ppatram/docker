# Video 27 of 59 - Kubernetes Network Policies

📺 https://www.youtube.com/watch?v=eVtnevr3Rao

## Overview

This video covers Kubernetes Network Policies — why they're needed, how CNI plugins enable them, and how to implement ingress/egress traffic restrictions between pods. The video demonstrates the default "all pods can communicate" behavior in Kubernetes, replacing the default Kind CNI (kindnet) with Calico (which supports network policies), and restricting front-end pod access to a database pod while allowing only backend pods to connect.

### Creating a Kind Cluster Without Default CNI

```bash
# Delete existing cluster
kind delete cluster --name cka

# Create cluster with custom config (CNI disabled)
kind create cluster --config kind.yaml --name cka-new
```

### Kind Config YAML (Disable Default CNI)

```yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
    extraPortMappings:
      - containerPort: 30001
        hostPort: 30001
  - role: worker
  - role: worker
networking:
  disableDefaultCNI: true
```

```yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4

  - role: control-plane
    extraPortMappings:
      - containerPort: 30001
        hostPort: 30001
  - role: worker
  - role: worker

  disableDefaultCNI: true
  podSubnet: "192.168.0.0/16"
```

### Manifest YAML (3-Tier Application)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: frontend
  labels:
    role: frontend
spec:
  containers:
    - name: nginx
      image: nginx
      ports:
        - containerPort: 80

apiVersion: v1

  name: frontend
spec:
  selector:
    role: frontend
  ports:
    - port: 80
      targetPort: 80
---

kind: Pod

  name: backend
  labels:
    role: backend
spec:
  containers:
    - name: nginx
      image: nginx
      ports:
        - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: backend
spec:
  selector:
    role: backend
  ports:
    - port: 80
      targetPort: 80
---
apiVersion: v1
kind: Pod

  name: mysql
  labels:
    name: mysql
spec:
  containers:
    - name: mysql
      image: mysql
      ports:
        - containerPort: 3306
      env:
        - name: MYSQL_ROOT_PASSWORD
          value: "password"

apiVersion: v1
kind: Service

  name: db
spec:
  selector:
    name: mysql
  ports:
    - port: 3306
      targetPort: 3306
```

### Testing Connectivity (Before Network Policy)

```bash
# Exec into frontend pod
kubectl exec -it frontend -- bash

# Test connectivity to backend

# Install telnet in frontend pod
apt-get update && apt-get install telnet -y

# Test connectivity to database
telnet db 3306

# Exit frontend pod
exit

# Exec into backend pod
kubectl exec -it backend -- bash

# Install telnet
apt-get update && apt-get install telnet -y

# Test connectivity to frontend
curl frontend

# Test connectivity to database
telnet db 3306

exit
```

### Applying Network Policy

```bash
# Apply network policy
kubectl apply -f netpolicy.yaml

# Describe network policy
kubectl describe netpol db-test

# Get network policies
kubectl get netpol
```

### Network Policy YAML (Restrict DB Access to Backend Only)

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: db-test
spec:
  podSelector:
    matchLabels:
      name: mysql
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              role: backend
      ports:
        - port: 3306
```

### Testing Connectivity (After Network Policy)

```bash
# Exec into frontend pod - should be BLOCKED to db
kubectl exec -it frontend -- bash
telnet db 3306
# Result: Connection times out (blocked)
exit

# Exec into backend pod - should be ALLOWED to db
kubectl exec -it backend -- bash
apt-get update && apt-get install telnet -y
telnet db 3306
# Result: Connected (allowed)
exit
```
