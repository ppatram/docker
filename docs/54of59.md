# Video 54 of 59 - Storage Classes and Dynamic Volume Provisioning

📺 https://www.youtube.com/watch?v=OKBTg2hMnw4

## Overview

This video covers Storage Classes and Dynamic Volume Provisioning in Kubernetes. It explains the difference between static provisioning (pre-creating PVs) and dynamic provisioning (PVs created on-demand via StorageClass). The demo is on GKE and shows: checking default storage classes, creating custom storage classes (high-performance SSD and cost-effective standard), changing the default storage class, creating PVCs with dynamic provisioning, and understanding volume binding modes (WaitForFirstConsumer vs Immediate).

## Terminal Sessions

### Check default storage classes

```bash
kubectl get storageclass
# or
kubectl get sc
```

### Describe a storage class

```bash
kubectl describe storageclass standard-rwo
```

### Create high-performance storage class

```bash
vi hp.yaml
kubectl apply -f hp.yaml
kubectl get storageclass
```

### Create cost-effective (dev) storage class

```bash
vi dev.yaml
kubectl apply -f dev.yaml
kubectl get storageclass
```

### Change the default storage class

```bash
kubectl patch storageclass cost-effective -p '{"metadata":{"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'
kubectl get storageclass
```

### Create PVC with high-performance storage class

```bash
vi pvc1.yaml
kubectl apply -f pvc1.yaml
kubectl get pvc
kubectl get pv
```

### Create PVC using default storage class (no storageClassName specified)

```bash
vi pvc2.yaml
kubectl apply -f pvc2.yaml
kubectl get pvc
kubectl get pv
```

### Describe PVC to check status

```bash
kubectl describe pvc prod-app-storage
kubectl describe pvc dev-app-storage
```

### Fix WaitForFirstConsumer issue - delete and recreate with Immediate binding

```bash
kubectl delete pvc prod-app-storage
kubectl delete storageclass high-performance
# Edit hp.yaml to change volumeBindingMode to Immediate
kubectl apply -f hp.yaml
kubectl apply -f pvc1.yaml
kubectl get pvc
kubectl get pv
```

## YAML Files

### High Performance Storage Class (hp.yaml)

```yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: high-performance
provisioner: pd.csi.storage.gke.io
parameters:
  type: pd-ssd
  replication-type: regional-pd
allowedTopologies:
  - matchLabelExpressions:
      - key: topology.gke.io/zone
        values:
          - us-central1-a
          - us-central1-b
volumeBindingMode: Immediate
reclaimPolicy: Retain
```

### Cost Effective Storage Class (dev.yaml)

```yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: cost-effective
provisioner: pd.csi.storage.gke.io
parameters:
  type: pd-standard
allowedTopologies:
  - matchLabelExpressions:
      - key: topology.gke.io/zone
        values:
          - us-central1-a
volumeBindingMode: Immediate
reclaimPolicy: Delete
```

### PVC with StorageClass (pvc1.yaml)

```yaml
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: prod-app-storage
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: high-performance
  resources:
    requests:
      storage: 50Gi
```

### PVC using default StorageClass (pvc2.yaml)

```yaml
apiVersion: v1

  accessModes:
    - ReadWriteOnce
```
