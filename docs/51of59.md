## Overview

This video covers Custom Resources (CR) and Custom Resource Definitions (CRD) in Kubernetes. It explains how to extend the Kubernetes API by creating new resource types beyond the default objects (pods, deployments, configmaps, etc.). The three main components are: CRD (the template), CR (an instance of the CRD), and a Custom Controller (manages the lifecycle of the CR). The video includes a hands-on demo creating a CRD, creating a custom resource, validating fields, and using `kubectl explain` to inspect CRD documentation. It also demonstrates installing cert-manager via Helm as a CRD example.

### Install cert-manager via Helm (CRD example)

```bash
helm repo add jetstack https://charts.jetstack.io
helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --create-namespace \
  --version v1.16.1 \
  --set crds.enabled=true
```

### Verify cert-manager CRDs

```bash
kubectl get crd | grep cert
kubectl get certificates
```

### Explain cert-manager certificate fields

```bash
kubectl explain certificates
kubectl explain certificates.spec.subject
```

## YAML Files

### Custom Resource Definition (CRD) - foos.samplecontroller.k8s.io

```yaml
apiVersion: apiextensions.k8s.io/v1
kind: CustomResourceDefinition
metadata:
  name: foos.samplecontroller.k8s.io
  annotations:
    # annotation details
spec:
  group: samplecontroller.k8s.io
  versions:
    - name: v1alpha1
      served: true
      storage: true
      schema:
        openAPIV3Schema:
          type: object
          properties:
            spec:
              type: object
              properties:
                deploymentName:
                  type: string
                replicas:
                  type: integer
                  minimum: 1
                  maximum: 10
            status:
              type: object
              properties:
                availableReplicas:
                  type: integer
  scope: Namespaced
  names:
    plural: foos
    singular: foo
    kind: Foo
```

### Custom Resource (CR) - example-foo

```yaml
apiVersion: samplecontroller.k8s.io/v1alpha1
kind: Foo
metadata:
  name: example-foo
spec:
  deploymentName: example-foo
  replicas: 2
```
