# Video 55 of 59 - CRI-DockerD Container Runtime Installation

📺 https://www.youtube.com/watch?v=b8BYiMawi8A

## Overview

This video covers the installation of the CRI-DockerD container runtime on a Kubernetes node. CRI-DockerD is the Container Runtime Interface shim for Docker, allowing Docker to be used as a container runtime with Kubernetes. The video demonstrates: verifying Docker is running, downloading and installing the CRI-DockerD Debian package, enabling and starting the service, configuring kernel-level system parameters (sysctl), and verifying the setup via logs. This is done on a killer.sh/killercoda playground matching the CKA exam environment.

## Terminal Sessions

### Verify cluster is healthy

```bash
kubectl get nodes
```

### Verify Docker is installed and running

```bash
systemctl status docker
```

### Download CRI-DockerD Debian package from GitHub releases

```bash
wget https://github.com/Mirantis/cri-dockerd/releases/download/v0.3.16/cri-dockerd_0.3.16.3-0.ubuntu-jammy_amd64.deb
```

### Install the CRI-DockerD package

```bash
sudo dpkg -i cri-dockerd_0.3.16.3-0.ubuntu-jammy_amd64.deb
```

### Enable and start CRI-DockerD service (persistent across reboots)

```bash
sudo systemctl enable --now cri-docker.service
sudo systemctl start cri-docker.service
```

### Verify the service is running

```bash
sudo systemctl status cri-docker.service
```

### Configure kernel-level system parameters (sysctl)

```bash
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward = 1
EOF
```

### Apply sysctl parameters

```bash
sudo sysctl --system
```

### Reload systemd daemon

```bash
sudo systemctl daemon-reload
```

### Check CRI-DockerD logs

```bash
journalctl -u cri-docker.service
```
