# Video 31/59 - DNS Fundamentals: How DNS Works

📺 [Watch on YouTube](https://www.youtube.com/watch?v=fDOoB4k4YSs)

## Overview

This video is a collaboration with Piyush Garg covering DNS (Domain Name System) fundamentals as a prerequisite for understanding CoreDNS in Kubernetes. Topics include: what DNS is and why it's needed, DNS resolution process, DNS caching at multiple levels (browser, OS, router, ISP), the 13 root name servers, TLD (Top Level Domain) servers, authoritative name servers, DNS record types (A, AAAA, CNAME, MX, NS, TXT), and important local DNS configuration files.

## Terminal Sessions

### DNS Lookup with nslookup

```bash
# Lookup a domain's IP address
nslookup thecloudopscommunity.org

# Use a specific DNS server (Cloudflare)
nslookup piyushgarg.dev 1.1.1.1

# Use Google's DNS server
nslookup piyushgarg.dev 8.8.8.8

# Lookup a CNAME record target
nslookup magic.piyushgarg.xyz
```

### Important DNS Configuration Files (Linux/macOS)

```bash
# View local DNS records (hosts file)
cat /etc/hosts

# View DNS resolver configuration
cat /etc/resolv.conf
```

### /etc/hosts file example

```
127.0.0.1   localhost
::1         localhost
127.0.0.1   teachers-local.site
```

### /etc/resolv.conf file example

```
# This shows which DNS server your system uses
# Common public DNS servers:
# Cloudflare: 1.1.1.1
# Google: 8.8.8.8
nameserver 192.168.1.1
```

## Key Concepts (No Additional Commands)

- **DNS Resolution Flow**: Browser → DNS Cache (browser/OS/router/ISP) → Root Name Servers (13) → TLD Servers (.com, .dev, .org) → Authoritative Name Server → IP Address returned
- **A Record**: Maps domain to IPv4 address
- **AAAA Record**: Maps domain to IPv6 address
- **CNAME Record**: Alias pointing to another domain (dynamic, follows the target's A record)
- **MX Record**: Mail exchange server for email routing
- **NS Record**: Delegates DNS resolution for a subdomain to another name server (port 53 UDP)
- **TXT Record**: Human-readable text, used for domain verification
- **DNS Caching**: Happens at browser, OS, router, and ISP levels to reduce latency
- **Root Name Servers**: 13 hardcoded IP addresses (internally load-balanced with anycast to many more servers)
