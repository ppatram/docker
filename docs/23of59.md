# Video 23 of 59 - Kubernetes Authorization and Kubeconfig

https://www.youtube.com/watch?v=P0bogYEyfeI

## Overview

This video covers Kubernetes authorization modes and the kubeconfig file. It explains how kubectl passes authentication details (server, client certificates, CA certificate, client key) through kubeconfig, the structure of kubeconfig (clusters, users, contexts), authorization modes (ABAC, RBAC, Node, Webhook, AlwaysAllow, AlwaysDeny), and how to inspect the kube-apiserver configuration to see authorization settings. It also shows where certificates are stored on the control plane node (`/etc/kubernetes/pki`).

## Key Concepts

- **Kubeconfig**: Contains clusters, users (with certificates), and contexts (user + cluster combination)
- **Default kubeconfig location**: `$HOME/.kube/config`
- **Authorization Modes**: ABAC, RBAC, Node, Webhook, AlwaysAllow, AlwaysDeny
- **ABAC**: Attribute-based, requires policy file and API server restart for changes
- **RBAC**: Role-based, uses Roles and RoleBindings, no restart needed
- **Node Authorization**: Used for node-to-node communication (e.g., API server to kubelet)
- **Webhook**: Third-party systems like OPA (Open Policy Agent)
- **Certificate storage**: `/etc/kubernetes/pki` on control plane node

## Terminal Sessions

### Viewing Kubeconfig

```bash
# View the kubeconfig file
vi $HOME/.kube/config

# View kubeconfig using kubectl
kubectl config view

# Specify a custom kubeconfig
kubectl --kubeconfig /path/to/config get pods

# Export kubeconfig as environment variable
export KUBECONFIG=/path/to/config
```

### Kubeconfig Structure (Sample)

```yaml
apiVersion: v1
kind: Config
clusters:
  - cluster:
      certificate-authority: /path/to/ca.crt
      server: https://cluster-server:6443
    name: my-cluster
users:
  - name: my-user
    user:
      client-certificate: /path/to/client.crt
      client-key: /path/to/client.key
contexts:
  - context:
      cluster: my-cluster
      user: my-user
    name: my-context
current-context: my-context
```

### Inspecting Kube API Server Configuration

```bash
# List running containers (for kind cluster)
docker ps | grep control

# Exec into the control plane node (kind cluster)
docker exec -it <control-plane-container-name> bash

# Navigate to static pod manifests
cd /etc/kubernetes/manifests

# List manifests
ls

# View kube-apiserver configuration
cat kube-apiserver.yaml
```

### Key API Server Parameters (from kube-apiserver.yaml)

```bash
# Authorization mode parameter in kube-apiserver
--authorization-mode=Node,RBAC

# Certificate locations
--client-ca-file=/etc/kubernetes/pki/ca.crt
--etcd-cafile=/etc/kubernetes/pki/etcd/ca.crt
--etcd-certfile=/etc/kubernetes/pki/apiserver-etcd-client.crt
--etcd-keyfile=/etc/kubernetes/pki/apiserver-etcd-client.key
```

### Viewing Certificate Files on Control Plane

```bash
# All certificates are stored in this directory
ls /etc/kubernetes/pki

# Contents include:
# ca.crt, ca.key - Root CA certificate and key
# apiserver.crt, apiserver.key - API server certificate and key
# apiserver-kubelet-client.crt, apiserver-kubelet-client.key - API server to kubelet client cert
# apiserver-etcd-client.crt, apiserver-etcd-client.key - API server to etcd client cert
```
