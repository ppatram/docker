## Overview

This video serves as the last-minute preparation guide for the CKA exam. It covers exam format, preparation strategy, tips and tricks for saving time, and sample exam tasks. The instructor shares their exam experience, including 15 of 17 tasks recalled from a recent exam sitting (July 2024). The exam runs on Ubuntu 20.04 with Kubernetes 1.30.

## Exam Tips & Key Points

- Use the preconfigured `k` alias for `kubectl` (already set in the exam sandbox)
- Bash auto-completion is available in the exam
- Use `Ctrl+Shift+C` and `Ctrl+Shift+V` for copy/paste in the Linux terminal
- Always set the correct context before starting any task
- Use imperative commands over declarative YAML when possible
- Read each question twice, complete all parts
- Use `sudo -i` for elevated access when required
- Carefully exit terminals (don't exit the default user session)
- Tackle easiest/quickest tasks first, bookmark complex ones for later

## Terminal Sessions

### Exam Strategy - Sample Commands Referenced

```bash
# Set kubectl alias (already configured in exam)
alias k=kubectl

# Enable bash auto-completion
source <(kubectl completion bash)
alias k=kubectl
complete -F __start_kubectl k

# Set context (provided before every question)
kubectl config use-context <context-name>

# Get elevated access
sudo -i
```

### Sample Task 1: Check schedulable nodes

```bash
# Check number of schedulable nodes excluding tainted (NoSchedule) nodes
kubectl get nodes
kubectl describe nodes | grep -i taint

# Write the count to a file
kubectl get nodes --no-headers | wc -l > /path/to/file
```

### Sample Task 2: Scale deployment

```bash
# Scale a deployment to 4 replicas (imperative)
kubectl scale deployment <deployment-name> --replicas=4

# Or edit the deployment directly
kubectl edit deployment <deployment-name>
```

### Sample Task 5: Expose deployment as NodePort service

```bash
kubectl expose deployment <deployment-name> --type=NodePort --port=80
```

### Sample Task 6: Monitor logs and filter

```bash
# Get logs from a pod, filter for specific string, redirect to file
kubectl logs <pod-name> | grep "error not found" > /path/to/file
```

### Sample Task 7: Find pod with highest CPU by label

```bash
# Get pods with specific label
kubectl get pods -l env=xyz

# Get pods sorted by CPU utilization
kubectl top pods -l env=xyz --sort-by=cpu
```

### Sample Task 8: Create multi-container pod

```bash
# Create multi-container pod (use dry-run then edit)
kubectl run multi-pod --image=redis --dry-run=client -o yaml > multi-pod.yaml
# Edit YAML to add second container with memcached image
kubectl apply -f multi-pod.yaml
```

### Sample Task 10: Fix unhealthy cluster (worker node not ready)

```bash
# SSH into the worker node
ssh worker-node

# Check kubelet status
sudo systemctl status kubelet

# Check kubelet logs
sudo journalctl -u kubelet

# Start kubelet and make changes permanent
sudo systemctl start kubelet
sudo systemctl daemon-reload
sudo systemctl enable kubelet
```

### Sample Task 11: Create ClusterRole, ClusterRoleBinding, and ServiceAccount

```bash
# Create service account
kubectl create serviceaccount <sa-name> -n test

# Create cluster role allowing deployments, services, daemonsets
kubectl create clusterrole <role-name> --verb=create --resource=deployments,services,daemonsets

# Create cluster role binding
kubectl create clusterrolebinding <binding-name> --clusterrole=<role-name> --serviceaccount=test:<sa-name>
```

### Sample Task 12: Mark node unschedulable and drain

```bash
# Cordon the node (mark unschedulable)
kubectl cordon <node-name>

# Drain the node (evict pods)
kubectl drain <node-name> --ignore-daemonsets --delete-emptydir-data
```

### Sample Task 13: Schedule pod on specific node

```bash
# Create pod with nodeSelector or nodeName
kubectl run <pod-name> --image=nginx --dry-run=client -o yaml > pod.yaml
# Edit YAML to add spec.nodeName: worker-01
kubectl apply -f pod.yaml
```

### Sample Task 15: Create PV, PVC, and Pod

```bash
# Create Persistent Volume (no storage class)
kubectl apply -f pv.yaml

# Create Persistent Volume Claim
kubectl apply -f pvc.yaml

# Create pod that uses the PVC
kubectl apply -f pod-with-pvc.yaml
```
