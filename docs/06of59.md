## Overview

This video provides an in-depth look at Kubernetes architecture, covering the control plane (master node) components and worker node components. Topics include: API server, scheduler, controller manager, etcd (key-value data store), kubelet, and kube-proxy. The video walks through the end-to-end flow of how a pod creation request is processed through the architecture.

## Key Concepts

- **Control Plane / Master Node**: Hosts administrative components (API server, scheduler, controller manager, etcd)
- **Worker Nodes**: Where actual workloads (pods/containers) run
- **Pod**: Smallest deployable unit in Kubernetes; encapsulates one or more containers
- **API Server**: Main entry point; authenticates, validates, and routes all requests
- **Scheduler**: Finds suitable nodes for pods based on resource constraints
- **Controller Manager**: Monitors and ensures all Kubernetes objects are healthy
- **etcd**: Key-value data store holding all cluster state information
- **Kubelet**: Node agent that receives instructions from API server and manages pods on the node
- **Kube-proxy**: Enables networking (pod-to-pod communication) via IP table rules

## Terminal Commands

> **Note:** This video is primarily conceptual, explaining Kubernetes architecture with diagrams. The only command mentioned is:

```bash
# User interacts with the cluster via kubectl
kubectl create pod <pod-details>
```

The video focuses on explaining the flow: User → kubectl → API Server → (authenticate/validate) → etcd → Scheduler → API Server → Kubelet → Pod created → Response back to user.

No hands-on terminal session is demonstrated in this video.
