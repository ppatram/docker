## Overview

This video covers JSON from a beginner's perspective, JSONPath as a query language to parse JSON data, and how to use JSONPath with advanced kubectl commands to query cluster data and workloads. Topics include understanding the difference between YAML and JSON output, querying specific fields using JSONPath expressions, using custom-columns for formatted output, and sorting results with `--sort-by`.

## Terminal Sessions

### Understanding JSON output from kubectl

```bash
# Get nodes in default human-readable format
kubectl get nodes

# Get nodes output in JSON format
kubectl get nodes -o json

# Get nodes output in YAML format
kubectl get nodes -o yaml
```

### Generating JSON and YAML files for comparison

```bash
# Generate a pod manifest in JSON format (dry-run)
kubectl run nginx --image=nginx --dry-run=client -o json > pod.json

# Generate a pod manifest in YAML format (dry-run)
kubectl run nginx --image=nginx --dry-run=client -o yaml > pod.yaml

# View both files side by side
cat pod.yaml
cat pod.json
```

### Creating a pod and querying with JSONPath

```bash
# Apply the pod manifest
kubectl apply -f pod.yaml

# Get pod in JSON format
kubectl get pods -o json

# Get pod labels using JSONPath
kubectl get pods -o=jsonpath='{.items[0].metadata.labels}'

# Get specific label value
kubectl get pods -o=jsonpath='{.items[0].metadata.labels.run}'

# Add newline formatting to JSONPath output
kubectl get pods -o=jsonpath='{.items[0].metadata.labels.run}{"\n"}'
```

### Querying container image and volume mount paths

```bash
# Get container image name
kubectl get pods -o=jsonpath='{.items[0].spec.containers[0].image}'

# Get volume mount path
kubectl get pods -o=jsonpath='{.items[0].spec.containers[0].volumeMounts[0].mountPath}'
```

### Using JSONPath with nodes

```bash
# Get all node names using wildcard
kubectl get nodes -o=jsonpath='{.items[*].metadata.name}'

# Get node names with newline formatting
kubectl get nodes -o=jsonpath='{.items[*].metadata.name}{"\n"}'
```

### Using custom-columns output

```bash
# Display node names with custom column header
kubectl get nodes -o=custom-columns=NODE:.metadata.name

# Display node names and IP addresses with custom columns
kubectl get nodes -o=custom-columns=NODE:.metadata.name,IP:.status.addresses[*].address
```

### Sorting output with --sort-by

```bash
# Create additional pods for sorting demo
kubectl run redis --image=redis
kubectl run aredis --image=redis

# Get pods
kubectl get pods

# Sort pods by name
kubectl get pods --sort-by=.metadata.name

# Sort pods by creation timestamp
kubectl get pods --sort-by=.metadata.creationTimestamp
```
