## Overview

This video covers the fundamentals of SSL/TLS encryption as a prerequisite for understanding certificates in Kubernetes. It explains symmetric vs asymmetric encryption, how HTTPS connections are established, the role of Certificate Authorities (CA), and how certificate signing requests (CSR) work. The video walks through how a user's symmetric key is securely transferred to a server using asymmetric encryption, and how CAs validate and sign certificates.

## Terminal Commands and Sessions


### Key Concepts (No Hands-on Lab)

- **Symmetric Encryption**: Same key used for encryption and decryption (vulnerable to interception)
- **Asymmetric Encryption**: Separate public/private key pair for encryption/decryption
- **SSH Authentication Flow**: Public key placed in `~/.ssh/authorized_keys` on server, private key retained by user
- **HTTPS Flow**: Server generates public/private key pair via OpenSSL, sends public key to client, client encrypts its symmetric key with server's public key, server decrypts with private key
- **Certificate Authority (CA)**: Validates domain ownership, signs certificates (e.g., DigiCert, Comodo)
- **Custom CA**: Used for internal/intranet domains within an organization
