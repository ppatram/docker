# Video 25 of 59 - Cluster Roles and Cluster Role Bindings

## Overview

This video explains the difference between namespace-scoped Roles/RoleBindings and cluster-scoped ClusterRoles/ClusterRoleBindings. It covers which Kubernetes resources are namespace-scoped vs cluster-scoped, how to create ClusterRoles and ClusterRoleBindings to grant access to cluster-level resources (like nodes), and demonstrates creating and testing cluster-level permissions for a user.

## Key Concepts

- **Namespace-scoped resources**: pods, deployments, replicasets, services, configmaps, secrets, roles, rolebindings
- **Cluster-scoped resources**: namespaces, nodes, clusterroles, clusterrolebindings, CSRs, storageclasses, persistentvolumes
- **Role / RoleBinding**: Namespace-scoped permissions (if no namespace specified, applies to all namespaces)
- **ClusterRole / ClusterRoleBinding**: Cluster-scoped permissions for cluster-level resources

## Terminal Sessions

### View Namespace-Scoped vs Cluster-Scoped Resources

```bash
# List all namespace-scoped resources
kubectl api-resources --namespaced=true

# List all cluster-scoped resources
kubectl api-resources --namespaced=false
```

### Check User Permissions for Cluster Resources

```bash
# Check if user krishna can get nodes
kubectl auth can-i get nodes --as krishna
```

### Create ClusterRole (Imperative)

```bash
# Create a cluster role with get, list, watch on nodes
kubectl create clusterrole node-reader --verb=get,list,watch --resource=nodes
```

### View ClusterRoles

```bash
# List all cluster roles (filter for your custom role)
kubectl get clusterroles | grep node-reader

# Describe the cluster role
kubectl describe clusterrole node-reader
```

### Create ClusterRoleBinding (Imperative)

```bash
# Create cluster role binding to attach cluster role to user
kubectl create clusterrolebinding reader-binding --clusterrole=node-reader --user=krishna
```

### View ClusterRoleBindings

```bash
# List cluster role bindings (filter for yours)
kubectl get clusterrolebinding | grep reader

# Describe the cluster role binding
kubectl describe clusterrolebinding reader-binding
```

### Test Cluster Role Permissions

```bash
# Check permission with auth can-i
kubectl auth can-i get nodes --as krishna

# Switch to user context
kubectl config get-contexts
kubectl config use-context krishna

# Get nodes (should work)
kubectl get nodes

# Describe nodes (should work - read access)
kubectl describe nodes

# Delete node (should FAIL - no delete permission)
kubectl delete node <node-name>
# Error: nodes is forbidden: User "krishna" cannot delete resource "nodes"
```

### Create ClusterRole Help

```bash
# View help for creating cluster roles
kubectl create clusterrole --help

# View help for creating cluster role bindings
kubectl create clusterrolebinding --help
```
