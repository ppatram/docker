# Video 35/59 - Upgrading a Kubernetes Cluster (kubeadm)

📺 [Watch on YouTube](https://www.youtube.com/watch?v=NtX75Ze47EU)

## Overview

This video covers the end-to-end process of upgrading a Kubernetes cluster using kubeadm. Topics include: drain, cordon, and uncordon concepts, Kubernetes version format (major.minor.patch), version skew policy (one minor version at a time), upgrade strategies (all-at-once, rolling update, blue-green), component version compatibility (API server must be highest), and a full hands-on demo upgrading from v1.29.6 to v1.30.2 on both control plane and worker nodes.

## Terminal Sessions

### Understanding Drain, Cordon, Uncordon

```bash
# Drain a node (evicts pods and marks unschedulable)
kubectl drain worker01 --ignore-daemonsets

# Cordon a node (mark as unschedulable without evicting)
kubectl cordon worker01

# Uncordon a node (mark as schedulable again)
kubectl uncordon worker01

# Check node status
kubectl get nodes
```

### Pre-Upgrade: Check OS and Available Versions

```bash
# Check OS version
cat /etc/os-release

# Update package list and check available kubeadm versions
sudo apt update
sudo apt-cache madison kubeadm
```

### Upgrade Control Plane Node

#### Step 1: Update Package Repository to Target Version

```bash
# Edit the Kubernetes package source to point to 1.30
sudo vi /etc/apt/sources.list.d/kubernetes.list
# Change v1.29 to v1.30

# Update apt cache
sudo apt update

# Verify available versions
sudo apt-cache madison kubeadm
```

#### Step 2: Upgrade kubeadm

```bash
# Install the target kubeadm version
sudo apt-mark unhold kubeadm
sudo apt-get update && sudo apt-get install -y kubeadm=1.30.2-1.1
sudo apt-mark hold kubeadm

# Verify kubeadm version
kubeadm version
```

#### Step 3: Plan and Apply the Upgrade

```bash
# View upgrade plan (dry run)
sudo kubeadm upgrade plan

# Apply the upgrade to control plane components
sudo kubeadm upgrade apply v1.30.2
```

#### Step 4: Drain the Control Plane Node

```bash
# Drain the master node
kubectl drain master --ignore-daemonsets
```

#### Step 5: Upgrade kubelet and kubectl on Control Plane

```bash
# Install new kubelet and kubectl versions
sudo apt-mark unhold kubelet kubectl
sudo apt-get update && sudo apt-get install -y kubelet=1.30.2-1.1 kubectl=1.30.2-1.1
sudo apt-mark hold kubelet kubectl

# Restart kubelet
sudo systemctl daemon-reload
sudo systemctl restart kubelet

# Verify kubelet status
sudo systemctl status kubelet
```

#### Step 6: Uncordon the Control Plane Node

```bash
# Make the master node schedulable again
kubectl uncordon master

# Verify
kubectl get nodes
```

### Upgrade Worker Nodes

#### Step 1: Update Package Repository (on worker node)

```bash
# Edit the Kubernetes package source
sudo vi /etc/apt/sources.list.d/kubernetes.list
# Change v1.29 to v1.30
```

#### Step 2: Upgrade kubeadm (on worker node)

```bash
sudo apt-mark unhold kubeadm
sudo apt-get update && sudo apt-get install -y kubeadm=1.30.2-1.1
sudo apt-mark hold kubeadm
```

#### Step 3: Upgrade Node Configuration (on worker node)

```bash
# Upgrade the local kubelet configuration
sudo kubeadm upgrade node
```

#### Step 4: Drain the Worker Node (from control plane)

```bash
# Drain the worker node (run from control plane)
kubectl drain worker01 --ignore-daemonsets
```

#### Step 5: Upgrade kubelet and kubectl (on worker node)

```bash
sudo apt-mark unhold kubelet kubectl
sudo apt-get update && sudo apt-get install -y kubelet=1.30.2-1.1 kubectl=1.30.2-1.1
sudo apt-mark hold kubelet kubectl

# Restart kubelet
sudo systemctl daemon-reload
sudo systemctl restart kubelet
```

#### Step 6: Uncordon the Worker Node (from control plane)

```bash
# Make the worker node schedulable again
kubectl uncordon worker01

# Verify all nodes
kubectl get nodes
```

### Repeat for Additional Worker Nodes

```bash
# Same steps for worker02:
# 1. Update package repo on worker02
# 2. Upgrade kubeadm on worker02
# 3. Run kubeadm upgrade node on worker02
# 4. Drain worker02 from control plane
kubectl drain worker02 --ignore-daemonsets
# 5. Upgrade kubelet and kubectl on worker02
# 6. Restart kubelet on worker02
sudo systemctl daemon-reload
sudo systemctl restart kubelet
# 7. Uncordon worker02 from control plane
kubectl uncordon worker02
```

### Verify Final State

```bash
# Check all nodes are upgraded and ready
kubectl get nodes

# Verify component versions
kubectl version
kubeadm version
kubelet --version
```

### Verify Control Plane Component Images

```bash
# Check the API server manifest for the updated image version
sudo cat /etc/kubernetes/manifests/kube-apiserver.yaml | grep image
```

## Key Concepts

- **Version Format**: major.minor.patch (e.g., 1.30.2)
- **Upgrade Policy**: One minor version at a time (1.28 → 1.29 → 1.30, cannot skip)
- **Support Policy**: Only 3 latest minor versions are supported at any time
- **Component Version Skew**:
  - kube-apiserver must be the highest version (X)
  - controller-manager, kube-scheduler: can be X-1
  - kubelet, kubectl: can be X-2
- **Upgrade Order**: Primary control plane → Additional control planes → Worker nodes
- **Upgrade Strategies**:
  - All-at-once (not recommended for production)
  - Rolling update (one node at a time - demonstrated)
  - Blue-green (provision new nodes with new version, swap out old nodes)
- **Drain vs Cordon**: Drain = evict pods + cordon; Cordon = mark unschedulable only
- **Important**: Pods not managed by a controller (Deployment/ReplicaSet) will be permanently lost when drained
- **kubeadm upgrade apply**: Upgrades control plane components (API server, controller manager, scheduler, proxy)
- **kubeadm upgrade node**: Upgrades local kubelet configuration on worker nodes
- kubelet and kubectl must be upgraded separately as system packages
