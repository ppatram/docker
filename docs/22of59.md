# Video 22 of 59 - TLS Certificates in Kubernetes: Creating and Approving CSRs

https://www.youtube.com/watch?v=LvPA-z8Xg4s

## Overview

This video explains how TLS/SSL works specifically in Kubernetes. It covers the different types of certificates (client, server, root), which components need certificates, how to differentiate between public certificates (`.crt`, `.pem` extensions) and private keys (`.key` extension or "key" in name), and walks through the complete process of creating a Certificate Signing Request (CSR) for a new user, approving it, and extracting the issued certificate.

## Key Concepts

- **Client Certificates**: Generated by clients (users, kube-scheduler, controller-manager, kube-proxy)
- **Server Certificates**: Generated by servers (kube-apiserver, etcd, kubelet)
- **Root Certificates**: Generated by the Certificate Authority
- **Naming Convention**: `.crt` or `.pem` = public certificate; `.key` or `-key` in name = private key
- Kube API Server acts as **server** for kubectl/scheduler/controller-manager and as **client** when talking to etcd/kubelet

## Terminal Sessions

### Step 1: Generate Private Key for New User

```bash
# Generate RSA private key for user "adam" (2048 bit)
openssl genrsa -out adam.key 2048
```

### Step 2: Generate Certificate Signing Request (CSR)

```bash
# Create CSR using the private key
openssl req -new -key adam.key -out adam.csr -subj "/CN=adam"
```

### Step 3: Encode CSR for Kubernetes CSR Object

```bash
# Encode the CSR in base64 and remove line breaks (required for YAML)
cat adam.csr | base64 | tr -d '\n'
```

### Step 4: Create Kubernetes CSR Object

```yaml
# csr.yaml
apiVersion: certificates.k8s.io/v1
kind: CertificateSigningRequest
metadata:
  name: adam
spec:
  request: <BASE64_ENCODED_CSR_FROM_STEP_3>
  expirationSeconds: 86400
  signerName: kubernetes.io/kube-apiserver-client
  usages:
    - client auth
```

### Step 5: Apply and Manage the CSR

```bash
# Apply the CSR YAML
kubectl apply -f csr.yaml

# View certificate signing requests
kubectl get csr

# Describe the CSR (shows status as Pending)
kubectl describe csr adam

# Approve the certificate signing request
kubectl certificate approve adam

# Verify approval
kubectl get csr
```

### Step 6: Extract the Issued Certificate

```bash
# Get the approved CSR as YAML and save to file
kubectl get csr adam -o yaml > issued_cert.yaml

# Decode the certificate from the issued CSR (extract the .status.certificate field)
echo "<CERTIFICATE_DATA_FROM_YAML>" | base64 --decode
```
