## Overview

Video 1 in the CKA series covering Docker fundamentals from scratch. Topics include: why containers are needed (environment misconfiguration problems), what containers are, containers vs virtual machines (house vs apartment building analogy), the Docker workflow (build, ship, run), Docker architecture (client, daemon, registry, runtime), and the basic Docker commands flow.

## Key Concepts

- Containers solve the "it works on my machine" problem by packaging app + dependencies + OS together
- Containers are lightweight sandboxes with only bare minimum libraries needed
- Docker is a platform to build, ship, and run containers (alternatives: Podman)
- Docker workflow: Dockerfile → `docker build` → Image → `docker push` → Registry → `docker pull` → `docker run` → Container
- Docker architecture: Client → Docker Daemon (dockerd) → Container Runtime
- Image registries: Docker Hub, Artifact Registry, JFrog Artifactory, Nexus
