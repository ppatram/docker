# Video 32/59 - CoreDNS: DNS in Kubernetes

📺 [Watch on YouTube](https://www.youtube.com/watch?v=VcWpZoRAQXE)

## Overview

This video covers how DNS works inside Kubernetes using CoreDNS. Topics include: demonstrating DNS resolution failures when CoreDNS is down, scaling CoreDNS back up to restore service name resolution, understanding the kube-dns service, examining `/etc/resolv.conf` inside pods, the CoreDNS ConfigMap (Corefile), and troubleshooting tips including the Calico networking add-on prerequisites (source/destination check on AWS, IP auto-detection).

## Terminal Sessions

### Testing DNS Resolution Between Pods/Services

```bash
# List pods in default namespace
kubectl get pods

# List services
kubectl get svc

# Exec into a pod and curl another service by name (fails if CoreDNS is down)
kubectl exec -it nginx -- curl nginx-one

# Curl by IP address (works even without DNS)
kubectl exec -it nginx -- curl <SERVICE_CLUSTER_IP>
```

### Diagnosing CoreDNS Issues

```bash
# Check CoreDNS pods in kube-system namespace
kubectl get pods -n kube-system

# Check CoreDNS deployment
kubectl get deploy -n kube-system

# Scale CoreDNS deployment back up
kubectl scale deploy coredns --replicas=2 -n kube-system

# Verify CoreDNS pods are running
kubectl get pods -n kube-system

# Verify DNS resolution now works
kubectl exec -it nginx -- curl nginx-one
```

### Examining DNS Configuration Inside a Pod

```bash
# Exec into a pod
kubectl exec -it nginx -- bash

# View DNS resolver config (inside the pod)
cat /etc/resolv.conf

# View hosts file (inside the pod)
cat /etc/hosts

# Get the pod's IP address
hostname -i
```

### /etc/resolv.conf inside a pod (example)

```
nameserver 10.96.0.10
search default.svc.cluster.local svc.cluster.local cluster.local
```

### Examining CoreDNS Service

```bash
# View the kube-dns service (CoreDNS is exposed as kube-dns)
kubectl get svc -n kube-system
```

### Examining CoreDNS ConfigMap

```bash
# List configmaps in kube-system
kubectl get configmap -n kube-system

# Describe the CoreDNS configmap
kubectl describe configmap coredns -n kube-system
```

### CoreDNS Corefile (from ConfigMap)

```yaml
# Corefile content
.:53 {
    errors
    health {
        lameduck 5s
    }
    ready
    kubernetes cluster.local in-addr.arpa ip6.arpa {
        pods insecure
        fallthrough in-addr.arpa ip6.arpa
    }
    prometheus :9153
    forward . /etc/resolv.conf
    cache 30
    loop
    reload
    loadbalance
}
```

### Checking Calico Networking (Prerequisite for CoreDNS)

kubectl get daemonset -n calico-system

### AWS-Specific Fix for Calico

```bash
# On AWS EC2: Disable source/destination check
# Go to: EC2 Console → Select Instance → Actions → Networking → Change source/destination check → Disable
```

## Key Concepts

- **CoreDNS** is the DNS server in Kubernetes, deployed as a Deployment in kube-system namespace
- **kube-dns** is the Kubernetes Service name for CoreDNS (exposed on port 53)
- Every pod gets an `/etc/resolv.conf` pointing to the kube-dns service IP (e.g., 10.96.0.10)
- CoreDNS requires a CNI network add-on (e.g., Calico, Cilium) to function
- The CoreDNS configuration is stored in a ConfigMap named `coredns` in the kube-system namespace
- The `cluster.local` is the default top-level domain for the Kubernetes cluster
