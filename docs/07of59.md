# Video 7: Installing Kubernetes Locally with Kind Cluster

📺 [Watch on YouTube](https://www.youtube.com/watch?v=RORhczcOrWs)

## Overview

This video covers setting up a local Kubernetes cluster using Kind (Kubernetes in Docker). It walks through installation, creating single-node and multi-node clusters, switching contexts between clusters, and introduces the `kubectl` CLI. The video also covers exam tips about accessing kubernetes.io/docs and the kubectl cheat sheet during the CKA exam.

## Terminal Sessions

### Installing Kind

```bash
# On macOS using Homebrew
brew install kind

# On Windows using Chocolatey
choco install kind

# On Linux - install from release binaries (check kind documentation)
```

### Creating a Single-Node Cluster

```bash
# Create cluster with specific Kubernetes version
kind create cluster --image kindest/node:v1.29.4 --name cka-cluster-1
```

### Verifying the Cluster

```bash
# Get cluster info
kubectl cluster-info --context kind-cka-cluster-1

# Check kubectl version
kubectl version --client

# List nodes
kubectl get nodes
```

### Installing kubectl

```bash
# On Linux - download binary via curl (see kubernetes.io/docs for full steps)
# On macOS/Windows - follow platform-specific instructions at kubernetes.io/docs
```

### Creating a Multi-Node Cluster with Config File

```bash
# Create the config file
vi config.yml
```

```yaml
# config.yml - Kind multi-node cluster configuration
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
  - role: worker
  - role: worker
```

```bash
# Create multi-node cluster using config file
kind create cluster --image kindest/node:v1.29.4 --name cka-cluster-2 --config config.yml

# Verify nodes (should show 3 nodes: 1 control-plane, 2 workers)
kubectl get nodes
```

### Switching Between Cluster Contexts

```bash
# List all available contexts
kubectl config get-contexts

# Switch to a specific cluster context
kubectl config use-context kind-cka-cluster-1

# Verify you're on the correct cluster
kubectl get nodes

# Switch to the other cluster
kubectl config use-context kind-cka-cluster-2

# Verify again
kubectl get nodes
```

## Exam Tips

- Before every question in the CKA exam, you must switch to the correct context (the command will be provided)
- Two websites accessible during the exam:
  - `kubernetes.io/docs` (and all subdomains)
  - `kubernetes.io/blog` (and all subdomains)
- The kubectl cheat sheet is found by searching "cheat sheet" on kubernetes.io/docs
