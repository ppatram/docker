# Video 9: Deployments and Replica Sets

📺 [Watch on YouTube](https://www.youtube.com/watch?v=oe2zjRb51F0)

## Overview

This video covers Replication Controllers, Replica Sets, and Deployments in Kubernetes. It explains how they ensure high availability by maintaining a desired number of pod replicas, load balancing, and how Deployments provide rolling updates and rollback capabilities. The video demonstrates creating each resource type, scaling replicas, and performing rolling updates.

## YAML Manifests

### Replication Controller

```yaml
# rc.yaml (initial version - Replication Controller)
apiVersion: v1
kind: ReplicationController
metadata:
  name: nginx-rc
  labels:
    env: demo
spec:
  template:
    metadata:
      labels:
        env: demo
    spec:
      containers:
        - name: nginx-container
          image: nginx
          ports:
            - containerPort: 80
  replicas: 3
```

### Replica Set

```yaml
# rc.yaml (updated to Replica Set)
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: nginx-rs
  labels:
    env: demo
spec:
  template:
    metadata:
      labels:
        env: demo
    spec:
      containers:
        - name: nginx-container
          image: nginx
          ports:
            - containerPort: 80
  replicas: 3
  selector:
    matchLabels:
      env: demo
```

### Deployment

```yaml
# rc.yaml (updated to Deployment)
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deploy
  labels:
    env: demo
spec:
  template:
    metadata:
      labels:
        env: demo
    spec:
      containers:
        - name: nginx-container
          image: nginx
          ports:
            - containerPort: 80
  replicas: 3
  selector:
    matchLabels:
      env: demo
```

## Terminal Sessions

### Replication Controller

```bash
# Check API version for replication controller
kubectl explain rc

# Apply the replication controller YAML
kubectl apply -f rc.yaml

# Verify pods are running (should show 3)
kubectl get pods

# Get replication controller status
kubectl get rc

# Describe a specific pod
kubectl describe pod nginx-rc-<random-id>
```

### Replica Set

```bash
# Check API version for replica set
kubectl explain rs

# Apply replica set (note: version is apps/v1, not v1)
kubectl apply -f rc.yaml

# Delete old replication controller
kubectl delete rc nginx-rc

# Verify pods
kubectl get pods
```

### Scaling Replicas

```bash
# Method 1: Update YAML file and apply
# (edit replicas field in rc.yaml from 3 to 5)
kubectl apply -f rc.yaml
kubectl get pods

# Method 2: Edit the live object directly
kubectl edit rs nginx-rs
# Change replicas in vi editor, save with :wq!
kubectl get pods

# Method 3: Imperative scale command
kubectl scale --replicas=10 rs/nginx-rs
kubectl get pods

# Get help on scale command
kubectl scale --help
```

### Deployment Operations

```bash
# Delete existing replica set
kubectl delete rs nginx-rs

# Apply deployment YAML
kubectl apply -f rc.yaml

# Verify
kubectl get pods
kubectl get deploy
kubectl get all
```

### Rolling Updates and Rollbacks

```bash
# Update image imperatively using set image
kubectl set image deploy/nginx-deploy \
  nginx-container=nginx:1.9.1

# Verify the image was updated
kubectl describe deploy nginx-deploy

# Check rollout history
kubectl rollout history deploy nginx-deploy

# Rollback to previous revision
kubectl rollout undo deploy nginx-deploy

# Verify rollback (check rollout history again)
kubectl rollout history deploy nginx-deploy

# Describe to confirm image reverted
kubectl describe deploy nginx-deploy
```

### Generating Deployment YAML from Dry Run

```bash
# Generate deployment YAML using dry-run
kubectl create deploy nginx-new --image=nginx --dry-run=client -o yaml > deploy.yaml
```

## Key Points

- **Replication Controller** (legacy) → **Replica Set** (preferred) → **Deployment** (most common)
- Replica Set adds `selector.matchLabels` to manage existing pods matching labels
- Replica Set uses `apiVersion: apps/v1` (not `v1`)
- Deployment manages Replica Sets, which manage Pods
- Deployments provide rolling updates (zero-downtime) and rollback capabilities
- Use `kubectl scale` for quick replica changes in the exam
- `vi` shortcut: `Shift+A` puts cursor at end of line in insert mode
