# Video 13 of 59 - DaemonSets, CronJobs, and Jobs

https://www.youtube.com/watch?v=kvITrySpy_k

## Overview

This video covers DaemonSets, CronJobs, and Jobs in Kubernetes. DaemonSets ensure one pod replica runs on every node in the cluster. CronJobs schedule tasks at specified intervals using cron syntax. Jobs run a task once to completion. Common DaemonSet use cases include monitoring agents, logging agents, kube-proxy, and CNI plugins (Weave, Flannel, Calico).

## Key Concepts

- DaemonSets create one pod per node (unlike Deployments which use a replicas field)
- When a node is added, DaemonSet automatically creates a pod on it
- When a node is removed, the DaemonSet pod is deleted
- Control plane nodes won't run DaemonSet pods by default (due to taints)
- CronJobs use standard cron syntax (minute, hour, day-of-month, month, day-of-week)
- Jobs run once and complete; CronJobs run on a schedule

## Terminal Sessions

### Creating a DaemonSet from a deployment YAML

```bash
# Edit the deployment YAML to convert it to a DaemonSet
# Change kind from Deployment to DaemonSet
# Remove the replicas field
# Keep apiVersion the same (apps/v1)

kubectl apply -f ds.yaml
kubectl get pods
# Shows 2 pods (one per worker node, not on control plane due to taints)
```

### DaemonSet YAML

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: ds
spec:
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx
        ports:
        - containerPort: 80
```

### Verifying DaemonSet behavior

```bash
# Check DaemonSet explain for API version
kubectl explain daemonset

# Delete a pod - it recreates automatically
kubectl delete pod <pod-name>
kubectl get pods

# Get DaemonSets across all namespaces
kubectl get ds -A
# or
kubectl get ds --namespace kube-system

# Shows kindnet (networking) and kube-proxy with 3 desired/ready
```

### Checking DaemonSets on a single-node cluster

```bash
# List kind clusters
kind get clusters

# Switch context to single-node cluster
kubectl config use-context kind-single-node

# Verify single node
kubectl get nodes

# Check DaemonSets - shows 1 desired/ready each
kubectl get daemonset --all-namespaces
kubectl get pods --all-namespaces
```

### Switching back to multi-node cluster

```bash
kubectl config use-context kind-cka-cluster-3
kubectl get nodes
```

### Describing a DaemonSet pod

```bash
kubectl describe pod <daemonset-pod-name>
# Shows "Controlled By: DaemonSet/ds"
```

### CronJob YAML example

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: hello
spec:
  schedule: "* * * * *"
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: hello
            image: busybox:1.28
            command:
            - /bin/sh
            - -c
            - date; echo Hello from the Kubernetes cluster
          restartPolicy: OnFailure
```

### Cron syntax reference

```bash
# Format: minute hour day-of-month month day-of-week
# * * * * *  = every minute
# 45 23 * * 6 = every Saturday at 11:45 PM
# */5 * * * * = every 5 minutes
# */10 * * * * = every 10 minutes
```
