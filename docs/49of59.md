# Video 49 of 59 — Pod Priority and Preemption in Kubernetes

https://www.youtube.com/watch?v=2lcx74HUz1U

## Overview

This video covers Pod Priority and Preemption, a topic newly added to the CKA 2025 curriculum. Topics include: how the Kubernetes scheduler handles resource-constrained environments, PriorityClass objects (value, globalDefault, preemptionPolicy), how higher-priority pods can evict lower-priority ones, creating and testing priority classes, observing preemption behavior with resource-constrained deployments, and analyzing events.

## Terminal Sessions

### Check existing PriorityClasses

```bash
kubectl get priorityclass
# or
kubectl get pc
```

### Explore PriorityClass structure

```bash
kubectl explain priorityclass
```

### Create PriorityClass YAML files

```bash
# Create high priority class
vi p1.yaml

# Create copies for medium and low
cp p1.yaml p2.yaml
cp p1.yaml p3.yaml

# Edit p2.yaml - change to medium-priority, value 10000, globalDefault false
vi p2.yaml

# Edit p3.yaml - change to low-priority, value 1000, globalDefault true
vi p3.yaml
```

### Apply PriorityClasses

```bash
# Apply all YAML files in current directory
kubectl apply -f .

# Verify
kubectl get pc
```

### Create a high-priority pod

```bash
vi pod1.yaml
kubectl apply -f pod1.yaml

# Verify
kubectl get pods
kubectl describe pod <pod-name>

# Clean up
kubectl delete pod <pod-name>
```

### Create resource-constrained deployment (low priority)

```bash
vi deploy.yaml
kubectl apply -f deploy.yaml

# Check node resources
kubectl get nodes
kubectl describe node controlplane

# Verify pods running
kubectl get pods
```

### Create a high-priority critical pod to trigger preemption

```bash
vi cr.yaml
kubectl apply -f cr.yaml

# Watch pods - observe preemption
kubectl get pods -w

# View events to see preemption details
kubectl get events
kubectl get events --sort-by=.metadata.creationTimestamp
```

### Observe preemption behavior

```bash
# Delete the critical pod
kubectl delete pod <critical-pod-name>

# Watch lower-priority pods recover
kubectl get pods

# Re-apply critical pod to observe preemption again
kubectl apply -f cr.yaml
kubectl get pods -w
```

### Cleanup

```bash
kubectl delete -f .
```

## YAML Files

### p1.yaml — High Priority Class

```yaml
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority
value: 1000000
globalDefault: false
description: "This priority should be used for high priority pods."
```

### p2.yaml — Medium Priority Class

```yaml
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: medium-priority
value: 10000
globalDefault: false
description: "This priority should be used for medium priority pods."
```

### p3.yaml — Low Priority Class (Global Default)

```yaml
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: low-priority
value: 1000
globalDefault: true
description: "This priority should be used for low priority pods."
```

### pod1.yaml — High Priority Pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: high-priority-nginx
spec:
  priorityClassName: high-priority
  containers:
    - name: nginx
      image: nginx
      resources:
        requests:
          memory: "128Mi"
          cpu: "500m"
```

### deploy.yaml — Resource-consuming low-priority deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: resource-hog
spec:
  replicas: 5
  selector:
    matchLabels:
      app: resource-hog
  template:
    metadata:
      labels:
        app: resource-hog
    spec:
      priorityClassName: low-priority
      containers:
        - name: nginx
          image: nginx
          resources:
            requests:
              memory: "256Mi"
              cpu: "500m"
```

### cr.yaml — Critical high-priority pod (triggers preemption)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: critical-nginx
spec:
  priorityClassName: high-priority
  containers:
    - name: nginx
      image: nginx
      resources:
        requests:
          memory: "256Mi"
          cpu: "15000m"
```
