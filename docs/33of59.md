# Video 33/59 - Kubernetes Networking: CNI, Container Runtime, and Pod Communication

📺 [Watch on YouTube](https://www.youtube.com/watch?v=EkAzMGldC5M)

## Overview

This is a collaboration with Saiyam Pathak (KubeSimplify) covering Kubernetes networking internals. Topics include: the history of Docker and container runtimes, OCI (Open Container Initiative) specifications, CRI (Container Runtime Interface), CNI (Container Network Interface) specification and implementations (Calico, Cilium, Flannel, Weave), the pause container and network namespaces, veth pairs, inter-node pod communication via bridges and ARC tables, and practical exploration of network namespaces on a Kubernetes node.

## Terminal Sessions

### Inspecting Pods and Network

```bash
# Get pods
kubectl get pods

# Get pods with wide output (shows IP and node)
kubectl get pods -o wide

# Get nodes
kubectl get nodes
```

### SSH into a Worker Node for Network Inspection

```bash
# SSH into the worker node
ssh node01
```

### Inspecting Network Namespaces on the Node

```bash
# List network namespaces
ip netns list

# List all namespaces for all processes
lsns

# Find the nginx process
lsns | grep nginx

# Show all namespaces for a specific process ID
lsns -p <PID>

# Execute inside a network namespace to see IP links
ip netns exec <namespace-name> ip link

# Show IP links on the host (shows veth pairs)
ip link show

# Show IP addresses on the host
ip addr
```

### Verifying Pod Network from kubectl

```bash
# Exec into the pod and check IP address
kubectl exec -it shared-namespace -- ip addr

# Exec into a pod and curl another pod by IP
kubectl exec -it nginx -- curl <POD_IP>
```

### Creating a Test Pod

## Key Concepts

- **OCI (Open Container Initiative)**: Defines image spec, runtime spec, and distribution spec for containers
- **Docker workflow**: docker run → Docker daemon → containerd → shim → runC (starts the container)
- **CRI (Container Runtime Interface)**: Allows kubelet to communicate with any container runtime (containerd, CRI-O)
- **CNI (Container Network Interface)**: Specification for configuring network in containers; implementations include Calico, Cilium, Flannel, Weave
- **CNI operations**: ADD, DELETE, CHECK, GC, VERSION
- **Pause container**: Created for every pod to hold the network namespace; enables containers in the same pod to share IP and communicate via localhost
- **veth pairs**: Virtual ethernet pairs connecting pod network namespace to the host's root network namespace
- **Bridge**: Resolves destination addresses using ARC tables for inter-pod communication on the same node
- **Network Policies**: Used to restrict traffic between pods/namespaces (important for CKA/CKS)
- **kube-proxy**: Manages IP table rules for service routing
- Nodes without a CNI plugin installed will remain in "NotReady" state
