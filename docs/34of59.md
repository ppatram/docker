# Video 34/59 - Ingress and Ingress Controllers in Kubernetes

📺 [Watch on YouTube](https://www.youtube.com/watch?v=kf3UjITS91M)

## Overview

This is a collaboration with Abhishek Veeramalla covering Kubernetes Ingress concepts. Topics include: why Ingress is needed (limitations of NodePort and LoadBalancer services), what Ingress is (inbound traffic routing), the relationship between Ingress Resource, Ingress Controller, and Load Balancer, deploying the NGINX Ingress Controller, creating Ingress rules for host-based and path-based routing, IngressClassName, and a full demo from building a Docker image to testing Ingress routing.

### Creating and Managing Ingress

```bash
# Apply ingress resource
kubectl apply -f ingress.yaml

# Check ingress (address may be empty without controller)
kubectl get ingress

# Delete ingress
kubectl delete -f ingress.yaml

# Re-apply ingress after controller is installed
kubectl apply -f ingress.yaml

# Check ingress again (address should populate)
kubectl get ingress
```

### Troubleshooting Ingress Controller

```bash
# Check ingress controller pods
kubectl get pods -A | grep nginx

# Check ingress controller service
kubectl get svc -n ingress-nginx

# View ingress controller pod configuration
kubectl edit pod <ingress-nginx-pod-name> -n ingress-nginx

# Check ingress controller logs
kubectl logs <ingress-nginx-pod-name> -n ingress-nginx

# Change service type from LoadBalancer to NodePort (if no cloud controller manager)
kubectl edit svc ingress-nginx-controller -n ingress-nginx
# Change type: LoadBalancer to type: NodePort
```

### Testing Ingress Routing

```bash
# Direct curl to ingress IP returns 404 (host header not matched)

# Curl with host resolution (simulates /etc/hosts entry)
curl --resolve example.com:80:<INGRESS_IP> http://example.com/
```

### Ingress YAML

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: hello-world-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx
  rules:
  - host: example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: hello-world-service
            port:
              number: 80
```

## Key Concepts

- **Ingress Resource**: Kubernetes YAML manifest defining routing rules (host-based, path-based)
- **Ingress Controller**: A Go program (written by load balancing companies) that watches Ingress resources and creates/configures load balancers
- **IngressClassName**: Specifies which Ingress Controller should watch a particular Ingress resource (prevents conflicts when multiple controllers are installed)
- **Flow**: DevOps deploys Ingress Controller → Creates Ingress Resource → Controller watches and creates Load Balancer → Load Balancer routes traffic to Services → Services route to Pods
- **Popular Ingress Controllers**: NGINX (community), NGINX (F5), Traefik, Kong, HAProxy, Ambassador
- **Without Cloud Controller Manager**: External IP stays "pending" for LoadBalancer services; workaround is to change service type to NodePort
- In CKA exam, the Ingress Controller is typically pre-installed
