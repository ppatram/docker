# Video 40 of 59 - Worker Node Failure Troubleshooting

📺 [Watch on YouTube](https://www.youtube.com/watch?v=U6PRwv7dJ-U)

## Overview

This video covers troubleshooting worker node failures in Kubernetes. It demonstrates diagnosing nodes in "NotReady" state by checking the kubelet service, verifying network add-ons (Calico), and fixing kubelet configuration issues. Key techniques include using `service kubelet status`, `journalctl`, and inspecting the kubelet config file at `/var/lib/kubelet/config.yaml`.

## Key Concepts

- If ALL nodes show NotReady, it's likely a network add-on issue (Calico, Flannel, Weave)
- If specific worker nodes show NotReady, check the kubelet service on that node
- kubelet is a systemd service (not a pod) — use `service` and `journalctl` commands
- kubelet config is at `/var/lib/kubelet/config.yaml`
- Network plugin config files are in `/etc/cni/net.d/`
- In the CKA exam, SSH into worker nodes to troubleshoot

## Terminal Sessions

### Checking node status and network add-on

```bash
kubectl get nodes
# master: Ready, worker1: NotReady, worker2: NotReady

# Check network add-on pods
kubectl get pods -A
kubectl get namespaces
# Look for calico-system, calico-apiserver, tigera-operator

kubectl get pods -n tigera-operator
kubectl get pods -n calico-system
kubectl get pods -n calico-apiserver
```

### Checking CNI configuration on a node

```bash
sudo su
cd /etc/cni/net.d/
ls
# Should see: 10-calico.conflist, calico-kubeconfig
exit
```

### Scenario 1: Worker node 1 - kubelet stopped

```bash
# SSH into worker node 1
ssh worker01

# Check kubelet status
sudo service kubelet status
# Active: inactive (dead), code=exited

# Start kubelet
sudo service kubelet start

# Verify kubelet is running
sudo service kubelet status
# Active: active (running)

# Back on control plane node
kubectl get nodes
# worker1: Ready
```

### Scenario 2: Worker node 2 - kubelet config error

```bash
# SSH into worker node 2
ssh worker02

# Check kubelet status
sudo service kubelet status
# Status: activating (auto-restart), Result: failure

# Check kubelet logs
journalctl -u kubelet
# Press Shift+G to go to last line
# Error: unable to load client CA file /etc/kubernetes/pki/WRONG.crt

# Check what files actually exist in the pki directory
ls /etc/kubernetes/pki/
# See correct file: ca.crt

# Edit kubelet config
sudo vi /var/lib/kubelet/config.yaml
# Fix: clientCAFile from "/etc/kubernetes/pki/WRONG.crt" to "/etc/kubernetes/pki/ca.crt"

# Restart kubelet
sudo service kubelet restart

# Verify kubelet is running
sudo service kubelet status
# Active: active (running)

# Back on control plane node
kubectl get nodes
# All nodes Ready
```

### Viewing kubelet service details (finding config paths)

```bash
sudo service kubelet status
# Shows ExecStart command with flags:
#   --bootstrap-kubeconfig=/etc/kubernetes/bootstrap-kubelet.conf
#   --kubeconfig=/etc/kubernetes/kubelet.conf
#   --config=/var/lib/kubelet/config.yaml
```
