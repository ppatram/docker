# Day 55 - Highly Available Kubernetes Cluster: Multiple Control Planes with HAProxy Load Balancer

https://www.youtube.com/watch?v=dGUAanaYr8o

## Overview

This is the final video of the CKA 2025 series covering how to set up a highly available Kubernetes cluster with:
- Multiple control plane (master) nodes (stacked etcd topology)
- Multiple worker nodes
- HAProxy load balancer to distribute API requests across control planes
- End-to-end setup on AWS EC2 instances
- Understanding control plane failure impact and mitigation

The architecture includes: 2 control plane nodes, 2 worker nodes, and 1 load balancer node. One master acts as the "leader" master where `kubeadm init` is run first.

### Master Node 1 (Leader) Setup

```bash
# SSH into master node 1
ssh -i <key.pem> ubuntu@<MASTER1_PUBLIC_IP>

# Elevate privileges
sudo -i

# Set prompt
PS1="master1 "

# Disable swap
swapoff -a
sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab

# Update kernel parameters for IPv4 forwarding and bridge traffic
cat <<EOF | tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

modprobe overlay
modprobe br_netfilter

cat <<EOF | tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

# Apply sysctl params without reboot
sysctl --system

# Verify settings
sysctl net.bridge.bridge-nf-call-iptables net.bridge.bridge-nf-call-ip6tables net.ipv4.ip_forward

# Install containerd (container runtime) v1.7.14
# Download and install containerd
wget https://github.com/containerd/containerd/releases/download/v1.7.14/containerd-1.7.14-linux-amd64.tar.gz
tar Cxzvf /usr/local containerd-1.7.14-linux-amd64.tar.gz

# Configure containerd as systemd service
mkdir -p /usr/local/lib/systemd/system
wget -O /usr/local/lib/systemd/system/containerd.service https://raw.githubusercontent.com/containerd/containerd/main/containerd.service

systemctl daemon-reload
systemctl enable --now containerd
systemctl status containerd

# Install runc
wget https://github.com/opencontainers/runc/releases/download/v1.1.12/runc.amd64
install -m 755 runc.amd64 /usr/local/sbin/runc

# Install CNI plugin
wget https://github.com/containernetworking/plugins/releases/download/v1.4.0/cni-plugins-linux-amd64-v1.4.0.tgz
mkdir -p /opt/cni/bin
tar Cxzvf /opt/cni/bin cni-plugins-linux-amd64-v1.4.0.tgz

# Install kubeadm, kubelet, kubectl
sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gpg

# Add Kubernetes apt repository
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.33/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.33/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

# Check kubelet status
systemctl status kubelet

# Check versions
kubelet --version
kubeadm version

# Initialize control plane with HA endpoint (leader master)
kubeadm init --control-plane-endpoint "<LB_PRIVATE_IP>:6443" --upload-certs --apiserver-advertise-address=<MASTER1_PRIVATE_IP>

# Configure kubectl for current user
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# Verify node (will be NotReady until CNI installed)
kubectl get nodes

# Install Calico network plugin
kubectl create -f https://raw.githubusercontent.com/projectcalico/calico/v3.28.0/manifests/tigera-operator.yaml
kubectl create -f https://raw.githubusercontent.com/projectcalico/calico/v3.28.0/manifests/custom-resources.yaml

# Watch pods come up
kubectl get pods -A
kubectl get pods -A -w

# Verify node is Ready
kubectl get nodes
```

## HAProxy Configuration File

```yaml
# Append to /etc/haproxy/haproxy.cfg

frontend kubernetes-api
    bind *:6443
    mode tcp
    default_backend kubernetes-master

backend kubernetes-master
    mode tcp
    balance roundrobin
    option tcp-check
    server master1 <MASTER1_PRIVATE_IP>:6443 check fall 3 rise 2
    server master2 <MASTER2_PRIVATE_IP>:6443 check fall 3 rise 2
```
