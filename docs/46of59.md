# Video 46 of 59 — Kustomize: Managing Kubernetes Deployments Across Multiple Environments

https://www.youtube.com/watch?v=Od4BROWS3M0

## Overview

This video covers Kustomize, a Kubernetes-native deployment tool that uses a layered approach on existing YAML files (unlike Helm which uses templating). Topics include: basic Kustomize usage with `kustomization.yaml`, importing resources, using `kubectl kustomize`, managing multiple environments with base/overlays directory structure, patches for environment-specific configuration (replicas, resource limits), config map generators, name prefixes/suffixes, and namespace overrides.

### Applying Kustomize configuration

```bash
# Preview the generated manifests
kubectl kustomize .

# Apply using kustomize with pipe
kubectl kustomize . | kubectl apply -f -

# Verify resources created
kubectl get all
kubectl get pods
```

### Creating per-directory kustomization files

```bash
# Create kustomization.yaml in each subdirectory (db, message-broker, enginex)
# Then update root kustomization.yaml to reference folders instead of individual files

# Apply changes from root directory
kubectl kustomize . | kubectl apply -f -

# Verify
kubectl get pods
kubectl get service
```


### Deploying dev environment

```bash
# Create dev namespace
kubectl create ns dev

# Apply dev overlay
kubectl kustomize /root/code/customize-enginex/overlays/dev | kubectl apply -f -

# Verify
kubectl get pods -n dev
kubectl get svc -n dev

# Test the service
curl <cluster-ip>
```

### Deploying test environment

```bash
# Create test namespace
kubectl create ns test

# Apply test overlay
kubectl kustomize /root/code/customize-enginex/overlays/test | kubectl apply -f -
```

```yaml
  - db/dbconfig.yaml

  - message-broker/rabbitmqservice.yaml
```

### base/deployment.yaml

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: enginex-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: enginex
  template:
    metadata:
      labels:
        app: enginex
    spec:
      containers:
        - name: enginex
          image: nginx:1.21
          ports:
            - containerPort: 80
          resources:
            requests:
              memory: "256Mi"
              cpu: "300m"
            limits:
              memory: "512Mi"
              cpu: "500m"
          volumeMounts:
            - name: html
              mountPath: /usr/share/nginx/html
      volumes:
        - name: html
          configMap:
            name: enginex-config
```

### base/service.yaml

```yaml
apiVersion: v1
kind: Service
metadata:
  name: enginex-service
spec:
  type: ClusterIP
  selector:
    app: enginex
  ports:
    - port: 80
      targetPort: 80
```

### base/kustomization.yaml

```yaml
resources:
  - deployment.yaml
  - service.yaml
configMapGenerator:
```

### overlays/dev/kustomization.yaml

```yaml
namePrefix: dev-
namespace: dev
resources:
  - ../../base
configMapGenerator:
  - name: enginex-config
    behavior: replace
    files:
      - index.html
patches:
  - path: replicas.yaml
  - path: limits.yaml
```
