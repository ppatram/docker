# Video 48 of 59 — Migrating Ingress to Gateway API in Kubernetes

https://www.youtube.com/watch?v=IICe8Z0lTa4

## Overview

This video demonstrates the manual migration of an Ingress resource to Gateway API. It covers: deploying a sample application with Ingress, installing an NGINX ingress controller, creating TLS secrets, setting up Gateway API CRDs, installing NGINX Gateway Fabric, creating GatewayClass/Gateway/HTTPRoute resources, handling ReferenceGrant for cross-namespace secrets, testing both HTTP and HTTPS routes, and finally deleting the old Ingress resource.

## Terminal Sessions

### Deploy the application

```bash
# Create namespace and deploy app (deployment, service, configmap)
kubectl create namespace web-app
kubectl apply -f - <<EOF
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-service
  namespace: web-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: web-service
  template:
    metadata:
      labels:
        app: web-service
    spec:
      containers:
        - name: nginx
          image: nginx
          ports:
            - containerPort: 80
          volumeMounts:
            - name: html
              mountPath: /usr/share/nginx/html
      volumes:
        - name: html
          configMap:
            name: web-config
---
apiVersion: v1
kind: Service
metadata:
  name: web-service
  namespace: web-app
spec:
  selector:
    app: web-service
  ports:
    - port: 80
      targetPort: 80
---
apiVersion: v1
kind: ConfigMap
metadata:
  name: web-config
  namespace: web-app
data:
  index.html: |
    <h1>Web Application Content</h1>
    <p>This is web front-end service.</p>
EOF

# Verify
kubectl get all -n web-app
kubectl get configmap -n web-app
kubectl get cm web-config -n web-app -o yaml
kubectl get deploy -n web-app -o yaml
```

### Generate self-signed TLS certificate

```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout tls.key -out tls.crt \
  -subj "/CN=gateway.web.local" \
  -addext "subjectAltName=DNS:gateway.web.local"

ls tls.*
```

### Create TLS secret

```bash
kubectl create secret tls web-tls-secret \
  --cert=tls.crt \
  --key=tls.key \
  -n web-app

kubectl get secret -n web-app
```

### Install NGINX Ingress Controller via Helm

```bash
helm install ingress-nginx ingress-nginx \
  --set controller.service.nodePorts.http=30082 \
  --set controller.service.nodePorts.https=30443 \
  --repo https://kubernetes.github.io/ingress-nginx
```

### Create Ingress resource

```bash
kubectl apply -f - <<EOF
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: web-ingress
  namespace: web-app
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "false"
spec:
  ingressClassName: nginx
  tls:
    - hosts:
        - gateway.web.local
      secretName: web-tls-secret
  rules:
    - host: gateway.web.local
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: web-service
                port:
                  number: 80
EOF

# Wait for address to be assigned
kubectl get ingress -n web-app
```

### Add DNS entry for Ingress

```bash
sudo vi /etc/hosts
# Add: <INGRESS_IP>  gateway.web.local
```

### Test Ingress

```bash
curl -k http://gateway.web.local
curl -k https://gateway.web.local
```

### Install Gateway API CRDs

```bash
kubectl apply -k "https://github.com/nginx/nginx-gateway-fabric/config/crd/gateway-api/standard?ref=v1.5.1"
```

### Install NGINX Gateway Fabric

```bash
kubectl apply -f https://raw.githubusercontent.com/nginx/nginx-gateway-fabric/v1.5.1/deploy/default/deploy.yaml
kubectl apply -f https://raw.githubusercontent.com/nginx/nginx-gateway-fabric/v1.5.1/deploy/nodeport/service.yaml

# Verify pods
kubectl get pods -n nginx-gateway
```

### Patch Gateway service ports

```bash
kubectl patch svc nginx-gateway-nginx-gateway-fabric -n nginx-gateway \
  -p '{"spec":{"ports":[{"port":80,"nodePort":30080,"protocol":"TCP","name":"http"},{"port":443,"nodePort":30081,"protocol":"TCP","name":"https"}]}}'

# Verify
kubectl get svc -n nginx-gateway
```

### Create GatewayClass

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: GatewayClass
metadata:
  name: nginx
spec:
  controllerName: gateway.nginx.org/nginx-gateway-controller
EOF

kubectl get gc
```

### Create Gateway resource

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: nginx-gateway
  namespace: nginx-gateway
spec:
  gatewayClassName: nginx
  listeners:
    - name: http
      protocol: HTTP
      port: 80
      hostname: gateway.web.local
      allowedRoutes:
        namespaces:
          from: All
    - name: https
      protocol: HTTPS
      port: 443
      hostname: gateway.web.local
      tls:
        mode: Terminate
        certificateRefs:
          - kind: Secret
            name: web-tls-secret
            namespace: web-app
      allowedRoutes:
        namespaces:
          from: All
EOF

kubectl get gateway -n nginx-gateway
```

### Create HTTPRoutes

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: web-route
  namespace: web-app
spec:
  parentRefs:
    - name: nginx-gateway
      namespace: nginx-gateway
      sectionName: http
  hostnames:
    - gateway.web.local
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: web-service
          port: 80
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: web-route-https
  namespace: web-app
spec:
  parentRefs:
    - name: nginx-gateway
      namespace: nginx-gateway
      sectionName: https
  hostnames:
    - gateway.web.local
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: web-service
          port: 80
EOF

kubectl get httproute -n web-app
```

### Create ReferenceGrant (for cross-namespace secret access)

```bash
kubectl apply -f - <<EOF
apiVersion: gateway.networking.k8s.io/v1beta1
kind: ReferenceGrant
metadata:
  name: allow-gateway-to-web-app-secret
  namespace: web-app
spec:
  from:
    - group: gateway.networking.k8s.io
      kind: Gateway
      namespace: nginx-gateway
  to:
    - group: ""
      kind: Secret
      name: web-tls-secret
EOF
```

### Verify and troubleshoot

```bash
kubectl describe gateway nginx-gateway -n nginx-gateway
kubectl describe httproute web-route -n web-app
kubectl describe httproute web-route-https -n web-app
```

### Update /etc/hosts for Gateway IP

```bash
kubectl get nodes -o wide
sudo vi /etc/hosts
# Add: <NODE_IP>  gateway.web.local
```

### Test Gateway API routing

```bash
# HTTP test using --resolve flag
curl --resolve gateway.web.local:30080:<NODE_IP> http://gateway.web.local:30080

# HTTPS test
curl -k https://gateway.web.local:30081

# Verbose to verify source
curl -kv https://gateway.web.local:30081
```

### Delete the old Ingress resource

```bash
kubectl get ingress -A
kubectl delete ingress web-ingress -n web-app
```
