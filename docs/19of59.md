# Video 19 of 59 - Health Probes in Kubernetes (Liveness, Readiness, Startup)

📺 https://www.youtube.com/watch?v=x2e6pIBLKzw

## Overview

This video covers Kubernetes Health Probes — Liveness, Readiness, and Startup probes. It demonstrates all three probe check types (command/exec, HTTP GET, TCP socket) with hands-on examples showing how liveness probes restart unhealthy containers, how readiness probes control traffic routing, and the key differences between each probe type.

## Terminal Sessions

### Liveness Probe - Command (exec) type

```bash
# Navigate to day 18 folder
cd day18/

# Apply liveness exec probe manifest
kubectl apply -f liveness-exec.yaml

# Watch pods for restarts
kubectl get pods --watch

# After ~30 seconds, the file is deleted and probe fails, triggering restart
# Describe pod to see probe failure
kubectl describe pod liveness-exec
# Shows: "Unhealthy: Liveness probe failed: cat: can't open '/tmp/healthy'"
# Shows: "Killing container with id... Liveness probe failed"
```

### Liveness Probe - HTTP GET type

```bash
# Apply liveness HTTP probe manifest
kubectl apply -f liveness-http.yaml

# Check pod status
kubectl get pods

# Describe pod to see HTTP probe failures
kubectl describe pod hello
# Shows: "Unhealthy: Liveness probe failed... 500" 
# Shows: container will be restarted

# Exec into the container to check the health endpoint
kubectl exec -it hello -- bash
ls /healthz
# File does not exist - that's why probe fails

# Try creating the file (container may restart before completion)
touch /healthz
```

### Liveness Probe - TCP Socket type

```bash
# Apply TCP liveness probe (port 8080)
kubectl apply -f liveness-tcp.yaml

# Check pod status - should stay running since port 8080 is open
kubectl get pods

# Describe pod
kubectl describe pod tcp-pod

# Modify YAML to probe port 3000 (not open) and force apply
kubectl apply -f liveness-tcp.yaml --force

# Check pods - will restart because port 3000 is not open
kubectl get pods

# Describe to see failure
kubectl describe pod tcp-pod
# Shows: "Unhealthy: Liveness probe failed: dial tcp... connection refused"
```

### Readiness Probe - HTTP GET type

```bash
# Apply liveness HTTP YAML with added readiness probe
kubectl apply -f liveness-http.yaml --force

# Check pods - container will show as not ready (0/1)
kubectl get pods

# Describe to see readiness probe failures
kubectl describe pod hello
# Shows: Readiness probe failed with 500 error
```

### Generate YAML output to view probe defaults

```bash
# View full pod spec to see default probe values
kubectl get pod tcp-pod -o yaml
# Shows failureThreshold: 3, successThreshold: 1, timeoutSeconds: 1
```

## YAML Manifests

### Liveness Probe - Command (exec) type

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: liveness-exec
spec:
  containers:
    - name: liveness
      image: busybox
      args:
        - /bin/sh
        - -c
        - touch /tmp/healthy; sleep 30; rm -f /tmp/healthy; sleep 600
      livenessProbe:
        exec:
          command:
            - cat
            - /tmp/healthy
        initialDelaySeconds: 5
        periodSeconds: 5
```

### Liveness Probe - HTTP GET type

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: hello
spec:
  containers:
    - name: liveness
      image: registry.k8s.io/liveness
      args:
        - /server
      livenessProbe:
        httpGet:
          path: /healthz
          port: 8080
        initialDelaySeconds: 3
        periodSeconds: 3
```

### Liveness Probe - TCP Socket type

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: tcp-pod
spec:
  containers:
    - name: liveness
      image: registry.k8s.io/liveness
      ports:
        - containerPort: 8080
      livenessProbe:
        tcpSocket:
          port: 8080
        initialDelaySeconds: 15
        periodSeconds: 10
```

### Combined Liveness and Readiness Probe

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: hello
spec:
  containers:
    - name: liveness
      image: registry.k8s.io/liveness
      args:
        - /server
      livenessProbe:
        httpGet:
          path: /healthz
          port: 8080
        initialDelaySeconds: 3
        periodSeconds: 3
      readinessProbe:
        httpGet:
          path: /healthz
          port: 8080
        initialDelaySeconds: 3
        periodSeconds: 3
```

## Key Concepts

- **Liveness Probe**: Monitors if the application is alive. Restarts the container on failure (after `failureThreshold` consecutive failures).
- **Readiness Probe**: Determines if the pod is ready to serve traffic. Does NOT restart the container — just removes it from the service endpoint.
- **Startup Probe**: For slow/legacy applications. Delays liveness and readiness probes until the application has fully started.
- **Three check types**: exec (command), httpGet, tcpSocket.
- **initialDelaySeconds**: Time to wait before the first probe check.
- **periodSeconds**: Interval between probe checks.
- **failureThreshold**: Number of consecutive failures before action (default: 3).
- **successThreshold**: Number of consecutive successes needed (default: 1).
- **Auto-added tolerations**: Kubernetes automatically adds `node.kubernetes.io/not-ready` and `node.kubernetes.io/unreachable` tolerations to pods for HA.
