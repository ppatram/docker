## Overview

This video provides a deep dive into Role-Based Access Control (RBAC) in Kubernetes. It covers creating roles and role bindings, checking user permissions, creating new users with certificates, setting up kubeconfig credentials and contexts for new users, troubleshooting expired certificates, and making REST API calls to the Kubernetes API with curl. The video demonstrates the complete workflow from user creation to verifying access control.

## Key Concepts

- **Core API Group**: `apiVersion: v1` (e.g., pods) — no group prefix
- **Named API Group**: `apiVersion: rbac.authorization.k8s.io/v1` (e.g., roles) — has group prefix
- **Role**: Defines permissions (verbs on resources) within a namespace
- **RoleBinding**: Associates a role with a user or group
- **Verbs**: get, list, watch, create, update, delete

## Terminal Sessions

### Check User Permissions

```bash
# Check if current user can get pods
kubectl auth can-i get pods

# Check which user you are
kubectl auth whoami

# Check if a specific user can get pods (impersonation)
kubectl auth can-i get pods --as adam

# Check if a specific user can get pods
kubectl auth can-i get pods --as krishna
```

### Create Role (Declarative)

```yaml
# role.yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: pod-reader
rules:
  - apiGroups: [""]
    resources: ["pods"]
    verbs: ["get", "watch", "list"]
```

```bash
# Apply the role
kubectl apply -f role.yaml

# List roles
kubectl get roles

# Describe a role
kubectl describe role pod-reader
```

### Create Role (Imperative)

```bash
# Create role imperatively
kubectl create role pod-reader --verb=create,get,list,update,delete --resource=pods
```

### Create RoleBinding (Declarative)

```yaml
# binding.yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: read-pods
subjects:
  - kind: User
    name: krishna
    apiGroup: rbac.authorization.k8s.io
roleRef:
  kind: Role
  name: pod-reader
  apiGroup: rbac.authorization.k8s.io
```

```bash
# Apply the role binding
kubectl apply -f binding.yaml

# List role bindings
kubectl get rolebinding

# Describe a role binding
kubectl describe rolebinding read-pods
```

### Create RoleBinding (Imperative)

```bash
# Create role binding imperatively
kubectl create rolebinding read-pods --role=pod-reader --user=krishna
```

### Generate New User Certificates (When Expired)

```bash
# Check certificate validity
openssl x509 -noout -dates -in krishna.crt

# Generate new private key
openssl genrsa -out krishna.key 2048

# Generate CSR from key
openssl req -new -key krishna.key -out krishna.csr -subj "/CN=krishna"

# Renew certificate using CA cert and key (from control plane node)
openssl x509 -req -in krishna.csr -CA ca.crt -CAkey ca.key -CAcreateserial -out krishna.crt -days 100
```

### Get CA Key from Control Plane Node (Kind Cluster)

```bash
# Find the control plane container
docker ps | grep control

# Exec into control plane
docker exec -it <cluster-control-plane-container> bash

# Certificates are in /etc/kubernetes/pki
cd /etc/kubernetes/pki

# View CA certificate
cat ca.crt

# View CA key
cat ca.key

# Exit container
exit
```

### Create CSR Object and Approve

```bash
# Encode CSR for YAML
cat krishna.csr | base64 | tr -d '\n'
```

```yaml
# csr.yaml
apiVersion: certificates.k8s.io/v1
kind: CertificateSigningRequest
metadata:
  name: krishna
spec:
  request: <BASE64_ENCODED_CSR>
  expirationSeconds: 8640000
  signerName: kubernetes.io/kube-apiserver-client
  usages:
    - client auth
```

```bash
# Apply CSR
kubectl apply -f csr.yaml

# View CSR status
kubectl get csr

# Approve CSR
kubectl certificate approve krishna

# Get issued certificate data
kubectl get csr krishna -o yaml

# Decode and save the certificate
kubectl get csr krishna -o jsonpath='{.status.certificate}' | base64 -d > krishna.crt
```

### Set Up Kubeconfig for New User

```bash
# Add user credentials to kubeconfig
kubectl config set-credentials krishna \
  --client-key=krishna.key \
  --client-certificate=krishna.crt \
  --embed-certs=true

# Create a context for the user
kubectl config set-context krishna \
  --cluster=kind-cka-cluster2 \
  --user=krishna

# View all contexts
kubectl config get-contexts

# Switch to the new user context
kubectl config use-context krishna

# Verify identity
kubectl auth whoami
```

### REST API Call with Curl

```bash
# Make a direct REST API call to Kubernetes API using certificates
curl -k https://<control-plane-ip>:<port>/api/v1/namespaces/default/pods \
  --key krishna.key \
  --cacert ca.crt \
  --cert krishna.crt
```

### Switch Between Contexts

```bash
# List contexts
kubectl config get-contexts

# Switch to admin context
kubectl config use-context <admin-context-name>

# Switch to user context
kubectl config use-context krishna
```
