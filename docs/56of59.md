# Day 54 - Pod Security: Security Context and Linux Capabilities

https://www.youtube.com/watch?v=2sk97jCQcP4

## Overview

This video covers pod security in Kubernetes, specifically Security Context and Linux Capabilities. Topics include:
- Security Context fields: `runAsUser`, `runAsGroup`, `runAsNonRoot`, `fsGroup`, `allowPrivilegeEscalation`
- Linux Capabilities: `CAP_NET_ADMIN`, `CAP_SYS_ADMIN`, `CAP_SYS_TIME`
- Adding and dropping capabilities
- Pod Security Standards: Privileged, Baseline, Restricted
- Applying Pod Security Standards to namespaces via labels
- Hands-on demos testing security contexts and capabilities

## Terminal Sessions

### Creating and testing Linux Capabilities pod

```bash
# Create capabilities test YAML
kubectl apply -f cap.yaml

# Check pod status
kubectl get pods

# Describe pod for troubleshooting
kubectl describe pod secure-demo

# Check logs
kubectl logs secure-demo

# Delete pod
kubectl delete pod secure-demo

# Apply updated YAML (with busybox image)
kubectl apply -f cap.yaml

# Force delete if needed
kubectl delete -f cap.yaml
kubectl delete pod secure-demo

# Re-apply
kubectl apply -f cap.yaml
kubectl get pods

# Exec into the pod
kubectl exec -it secure-demo -- sh
```

### Inside the capabilities pod (testing NET_ADMIN capability)

```bash
# Show network interfaces
ip link show

# Create a dummy network interface (tests NET_ADMIN capability)
ip link add dummy0 type dummy

# Verify dummy interface was created
ip link show

# Test SYS_TIME capability - check current date
date

# Attempt to set date
date -s "$date"

# Try to mount filesystem (should fail - no SYS_ADMIN capability)

```

### Testing Pod Security Standards with namespaces

```bash
# Create restricted namespace
kubectl create ns restricted-ns
kubectl get ns

# Apply Pod Security Standard labels to namespace
kubectl label namespace restricted-ns \
  pod-security.kubernetes.io/enforce=restricted \
  pod-security.kubernetes.io/audit=restricted \
  pod-security.kubernetes.io/warn=restricted

# Verify labels
kubectl get ns --show-labels

# Apply compliant pod
kubectl apply -f pod1.yaml

# Check pod in restricted namespace
kubectl get pod -n restricted-ns

# Check logs if crash
kubectl logs compliant-pod -n restricted-ns

# Delete and re-apply with busybox image
kubectl delete pod compliant-pod -n restricted-ns
kubectl apply -f pod1.yaml
kubectl get pod -n restricted-ns

# Apply non-compliant pod (should be rejected)
kubectl apply -f pod2.yaml

# Check events in namespace
kubectl get events -n restricted-ns
```

## YAML Files

### cap.yaml - Capabilities test pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-demo
spec:
  containers:
  - name: app
    image: busybox:1.28
    command: ["sh", "-c", "sleep 3600"]
    securityContext:
      capabilities:
        drop:
          - ALL
        add:
          - NET_ADMIN
          - SYS_TIME
```

### sec.yaml - Security Context test pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-demo
spec:
  securityContext:
    runAsUser: 1000
    runAsGroup: 3000
    runAsNonRoot: true
    fsGroup: 2000
  volumes:
  - name: demo-vol
    emptyDir: {}
  containers:
  - name: app
    image: busybox:1.28
    command: ["sh", "-c", "sleep 3600"]
    volumeMounts:
    - name: demo-vol
      mountPath: /data/demo
    securityContext:
      runAsUser: 1000
      runAsNonRoot: true
      allowPrivilegeEscalation: false
      capabilities:
        drop:
          - ALL
```

### root.yaml - Root user test pod

apiVersion: v1

### pod1.yaml - Compliant pod for restricted namespace

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: compliant-pod
  namespace: restricted-ns
spec:
  securityContext:
    runAsUser: 1000
    runAsGroup: 1000
    runAsNonRoot: true
    fsGroup: 1000
    seccompProfile:
      type: RuntimeDefault
  containers:
  - name: app
    image: busybox:1.28
    command: ["sh", "-c", "sleep 3600"]
    securityContext:
      allowPrivilegeEscalation: false
      runAsNonRoot: true
      runAsUser: 1000
      capabilities:
        drop:
          - ALL
```

### pod2.yaml - Non-compliant pod (violates restricted standard)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: non-compliant-pod
  namespace: restricted-ns
spec:
  containers:
  - name: app
    image: busybox:1.28
    command: ["sh", "-c", "sleep 3600"]
    securityContext:
      privileged: true
      runAsUser: 0
      allowPrivilegeEscalation: true
      capabilities:
        add:
          - NET_ADMIN
          - SYS_ADMIN
    volumeMounts:
    - name: host-vol
      mountPath: /host
  volumes:
  - name: host-vol
    hostPath:
      path: /
```
