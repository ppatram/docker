## Overview

This video introduces Helm, the package manager for Kubernetes. It covers what Helm is, why it's needed, and its three main components: Charts (packaged bundles of Kubernetes manifests), Repositories (where charts are stored, e.g., artifacthub.io), and Releases (running instances of charts). The video includes a hands-on demo installing Helm, creating a sample chart, exploring chart structure, installing ArgoCD from a Helm repository, and managing releases with upgrade, rollback, search, and pull commands.

## Terminal Sessions

### Verify cluster is ready

```bash
kubectl get nodes
```

### Install Helm on Ubuntu

```bash
curl https://baltocdn.com/helm/signing.asc | gpg --dearmor | sudo tee /usr/share/keyrings/helm.gpg > /dev/null
sudo apt-get install apt-transport-https --yes
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/helm.gpg] https://helm.sh/debian/ all main" | sudo tee /etc/apt/sources.list.d/helm-stable-debian.list
sudo apt-get update
sudo apt-get install helm

# Verify installation
which helm
helm version
```

### If helm is installed in wrong path

```bash
sudo cp /usr/local/bin/helm /usr/bin/helm
which helm
helm version
```

### Helm help and basic commands

```bash
helm --help
helm create --help
```

### Create a sample Helm chart

```bash
helm create sample-chart

ls
cd sample-chart

# View chart structure
find .
ls

# View chart.yaml (chart metadata)
cat Chart.yaml

# View values.yaml (configurable values)
cat values.yaml

# View a template file (parameterized deployment)
cat templates/deployment.yaml

# View helper template
cat templates/_helpers.tpl
```

### Add a Helm repository and install ArgoCD

```bash
# Add the Argo Helm repository
helm repo add argo https://argoproj.github.io/argo-helm

# Install ArgoCD from the repo
helm install my-argo-cd argo/argo-cd --version 7.8.10

# Verify the release
helm list
```

### Port forward to access ArgoCD (optional)

```bash
kubectl port-forward service/my-argo-cd-argocd-server -n default 28080:443
```

### Upgrade a Helm release

```bash
helm upgrade my-argo-cd argo/argo-cd

# Verify - shows new revision
helm list
```

### Pull a chart locally

```bash
helm pull argo/argo-cd

# Extract the chart
ls
tar xvf argo-cd-*.tgz

# Explore the extracted chart
ls argo-cd/
cat argo-cd/values.yaml
cat argo-cd/templates/deployment.yaml
```

### Install from a local chart directory

```bash
# Install using local chart directory
helm install my-release ./argo-cd --version <version>
```

### Rollback a Helm release

```bash
helm rollback my-argo-cd

# Verify - shows new revision with rollback
helm list
```

### Search for charts

```bash
# Search in Artifact Hub
helm search hub argo

# Search in locally added repos
helm search repo argo
```

## Helm Chart Structure

```
sample-chart/
├── Chart.yaml          # Chart metadata (name, version, appVersion)
├── values.yaml         # Default configurable values
├── charts/             # Chart dependencies
└── templates/          # Kubernetes manifest templates
    ├── deployment.yaml
    ├── service.yaml
    ├── serviceaccount.yaml
    ├── hpa.yaml
    ├── ingress.yaml
    ├── NOTES.txt
    ├── _helpers.tpl    # Template helper functions
    └── tests/
        └── test-connection.yaml
```

## Key Helm Concepts

| Concept | Description |
|---------|-------------|
| **Chart** | A package/bundle of templated Kubernetes YAML files |
| **Repository** | A location where charts are stored (e.g., artifacthub.io) |
| **Release** | A running instance of a chart deployed to a cluster |
| **values.yaml** | File containing configurable values referenced by templates |
| **Chart.yaml** | Metadata file with chart name, version, and app version |
| **templates/** | Directory containing parameterized Kubernetes manifests |

## YAML Files

### Chart.yaml (example)

```yaml
apiVersion: v2
name: sample-chart
description: A Helm chart for Kubernetes
type: application
version: 0.1.0
appVersion: "1.16.0"
```

### values.yaml (example)

```yaml
replicaCount: 1

image:
  repository: nginx
  pullPolicy: IfNotPresent
  tag: ""

serviceAccount:
  create: true
  name: ""

service:
  type: ClusterIP
  port: 80
```
