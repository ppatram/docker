# Video 15 of 59 - Taints and Tolerations, and Node Selectors

https://www.youtube.com/watch?v=nwoS2tK2s6Q

## Overview

This video covers taints and tolerations in Kubernetes, which control which pods can be scheduled on which nodes. Taints are applied to nodes to repel pods that don't tolerate the taint. Tolerations are applied to pods to allow scheduling on tainted nodes. The video also covers node selectors, which allow pods to choose which node to run on based on node labels. The video distinguishes between these approaches: taints/tolerations restrict what pods a node accepts, while node selectors let pods choose their node.

## Key Concepts

- **Taints** are applied to nodes — they repel pods without matching tolerations
- **Tolerations** are applied to pods — they allow scheduling on tainted nodes
- **Effects**: `NoSchedule` (new pods only), `NoExecute` (evicts existing + blocks new), `PreferNoSchedule` (best effort, no guarantee)
- Control plane nodes have a taint by default to prevent custom workloads
- Taints/tolerations restrict which pods a node accepts but don't guarantee placement
- **Node Selectors** let pods choose their node based on labels (the pod decides)
- Node affinity/anti-affinity (covered in next video) provides more advanced scheduling

## Terminal Sessions

### Initial cleanup

```bash
kubectl delete pod nginx redis
```

### Viewing nodes

```bash
kubectl get nodes
```

### Applying taints to nodes

```bash
# Taint worker node 1
kubectl taint node cka-cluster-3-worker gpu=true:NoSchedule

# Taint worker node 2
kubectl taint node cka-cluster-3-worker2 gpu=true:NoSchedule
```

### Verifying taints

```bash
kubectl describe node cka-cluster-3-worker | grep -i Taints
# Shows: gpu=true:NoSchedule
```

### Testing pod scheduling with taints

```bash
# Try to schedule a pod - will be Pending
kubectl run nginx --image=nginx
kubectl get pod
# Status: Pending

# Describe shows why it can't schedule
kubectl describe pod nginx
# 0/3 nodes available: 1 has taint node-role.kubernetes.io/control-plane,
# 2 have untolerated taint gpu=true
```

### Creating a pod with toleration

```bash
# Generate YAML for a new pod
kubectl run redis --image=redis --dry-run=client -o yaml > redis.yaml
```

### Pod YAML with toleration

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: redis
  labels:
    run: redis
spec:
  containers:
  - name: redis
    image: redis
  tolerations:
  - key: "gpu"
    operator: "Equal"
    value: "true"
    effect: "NoSchedule"
```

```bash
kubectl apply -f redis.yaml
kubectl get pods
# redis is Running

kubectl get pods -o wide
# redis is scheduled on worker node 1 (can tolerate the taint)
# nginx is still Pending (cannot tolerate)
```

### Removing a taint from a node

```bash
# Check current taint
kubectl describe node cka-cluster-3-worker2 | grep -i Taints

# Remove taint by appending a hyphen at the end
kubectl taint node cka-cluster-3-worker2 gpu=true:NoSchedule-

# Verify taint is removed
kubectl describe node cka-cluster-3-worker2 | grep -i Taints
# Shows: <none>

# nginx pod now gets scheduled on the untainted node
kubectl get pods
kubectl get pods -o wide
# nginx now running on worker2
```

### Node Selectors - scheduling pods based on node labels

```bash
# Generate pod YAML
kubectl run nginx-new --image=nginx --dry-run=client -o yaml > new-nginx.yaml
```

### Pod YAML with nodeSelector

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-new
  labels:
    run: nginx-new
spec:
  containers:
  - name: nginx-new
    image: nginx
  nodeSelector:
    gpu: "false"
```

```bash
# Apply - pod will be Pending (no node has the matching label)
kubectl apply -f new-nginx.yaml
kubectl get pods
# nginx-new: Pending

kubectl describe pod nginx-new
# 2 nodes did not match pod's node affinity/selector
```

### Labeling a node

```bash
kubectl get nodes

# Add label to a node
kubectl label node cka-cluster-3-worker gpu=false

# Verify label
kubectl get nodes --show-labels
# Shows gpu=false on worker node
```

### Verifying node selector scheduling

```bash
# Delete and recreate pod to trigger scheduling
kubectl delete pod nginx-new
kubectl apply -f new-nginx.yaml

kubectl get pods
# nginx-new: Running

kubectl get pods -o wide
# Scheduled on cka-cluster-3-worker (the labeled node)
```
