# Video 53 of 59 - Admission Controllers in Kubernetes

📺 https://www.youtube.com/watch?v=wVWV7keQEV4

## Overview

This video covers Admission Controllers in Kubernetes — pieces of code that intercept API server requests after authentication and authorization. It explains the two types: Mutating Admission Webhooks (which modify objects) and Validating Admission Webhooks (which accept/reject requests). The demo shows how to check enabled admission plugins, enable the NamespaceAutoProvision plugin, test ResourceQuota validation (validating controller), and test LimitRange defaults (mutating controller).

## Terminal Sessions

### Check cluster nodes

```bash
kubectl get nodes
```

### Check API server pod

```bash
kubectl get pods -n kube-system
```

### View API server YAML and check admission plugins

```bash
kubectl get pods -n kube-system kube-apiserver-controlplane -o yaml | grep admission
```

### Check the kube-apiserver manifest on the control plane

```bash
ls /etc/kubernetes/manifests/
cat /etc/kubernetes/manifests/kube-apiserver.yaml | grep admission
```

### Test creating a pod in a non-existent namespace (before enabling plugin)

```bash
kubectl run nginx --image=nginx -n test
# Error: namespace "test" not found
kubectl get namespace
```

### Backup and edit the API server manifest to enable NamespaceAutoProvision

```bash
cd /etc/kubernetes/manifests/
cp kube-apiserver.yaml kube-apiserver_backup.yaml
vi kube-apiserver.yaml
# Add NamespaceAutoProvision to --enable-admission-plugins=NodeRestriction,NamespaceAutoProvision
```

### Wait for API server to restart

```bash
kubectl get pods -n kube-system
# May get "connection refused" while restarting
```

### Force restart by moving manifest and moving it back

```bash
mv /etc/kubernetes/manifests/kube-apiserver.yaml /tmp/
mv /etc/kubernetes/manifests/etcd.yaml /tmp/
# Wait for connection refused
mv /tmp/kube-apiserver.yaml /etc/kubernetes/manifests/
mv /tmp/etcd.yaml /etc/kubernetes/manifests/
# Wait for API server to come back up
```

### Verify admission plugin is enabled

```bash
kubectl get pods -n kube-system kube-apiserver-controlplane -o yaml | grep admission
```

### Test NamespaceAutoProvision (mutating admission controller)

```bash
kubectl get ns
kubectl run nginx --image=nginx -n test
# Pod created successfully
kubectl get ns
# "test" namespace auto-created

kubectl run nginx2 --image=nginx -n pu
# Pod created
kubectl get ns
# "pu" namespace auto-created
```

### Create ResourceQuota demo (validating admission controller)

```bash
kubectl apply -f demo.yaml
kubectl get ns
kubectl get resourcequota -A
```

### Attempt to create pod exceeding quota

```bash
kubectl apply -f pod.yaml
# Error: forbidden - failed quota, must specify limits for CPU and memory
```

### Create LimitRange (mutating admission controller)

```bash
kubectl apply -f limits.yaml
kubectl get limitrange -n quota-demo
```

### Create pod without limits/requests and verify defaults applied

```bash
kubectl run nginx --image=nginx -n quota-demo
kubectl get pods -n quota-demo
kubectl describe pod nginx -n quota-demo
# Limits and requests automatically applied by LimitRange
```

## YAML Files

### ResourceQuota and Namespace (demo.yaml)

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: quota-demo
---
apiVersion: v1
kind: ResourceQuota
metadata:
  name: quota-demo
  namespace: quota-demo
spec:
  hard:
    requests.cpu: "1"
    requests.memory: 1Gi
    limits.cpu: "2"
    limits.memory: 2Gi
    pods: "4"
```

### Pod exceeding quota (pod.yaml)

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: big-pod
  namespace: quota-demo
spec:
  containers:
    - name: nginx
      image: nginx
      resources:
        requests:
          memory: "2Gi"
          cpu: "1"
```

### LimitRange (limits.yaml)

```yaml
apiVersion: v1
kind: LimitRange
metadata:
  name: default-limits
  namespace: quota-demo
spec:
  limits:
    - type: Container
      default:
        memory: "512Mi"
        cpu: "200m"
      defaultRequest:
        memory: "256Mi"
        cpu: "100m"
```
