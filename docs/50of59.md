# Video 50 of 59 — StatefulSets: Deploying Stateful Applications in Kubernetes

https://www.youtube.com/watch?v=roc8gzDeSmE

## Overview

This video covers StatefulSets in Kubernetes — how they differ from Deployments (stable network identity, ordinal indexing, sequential pod creation/deletion), and when to use them for stateful applications. The demo deploys a MongoDB application as a StatefulSet with headless service, StorageClass, PersistentVolumes, and PersistentVolumeClaims. Topics include: scaling StatefulSets, verifying data persistence across pod restarts, and the one-to-one mapping between pods and persistent storage.

## Terminal Sessions

### Verify cluster

```bash
kubectl get nodes
kubectl cluster-info
```

### Prepare worker node storage directories

```bash
ssh node01

# Create mount directories for MongoDB pods (0 through 4)
mkdir -p /mnt/data/mongodb-{0,1,2,3,4}
ls /mnt/data/

exit
```

### Create headless service for StatefulSet

```bash
vi mongo-svc.yaml
kubectl apply -f mongo-svc.yaml

# Verify - note ClusterIP is "None"
kubectl get svc
```

### Create StorageClass

```bash
vi sc.yaml
kubectl apply -f sc.yaml
kubectl get sc
```

### Create PersistentVolumes

```bash
vi pv.yaml
kubectl apply -f pv.yaml

# Verify 5 PVs created
kubectl get pv
```

### Create StatefulSet

```bash
vi ss.yaml
kubectl apply -f ss.yaml

# Watch sequential pod creation (0, then 1, then 2)
kubectl get statefulset
kubectl get pods -w
kubectl get pods

# Verify PV binding
kubectl get pv
kubectl get pvc
```

### Test data persistence

```bash
# Exec into the first MongoDB pod
kubectl exec -it mongodb-0 -- mongosh

# Inside mongo shell:
use testDB
db.users.insert({name: "user1", email: "user1@example.com"})
db.users.insert({name: "user2", email: "user2@example.com"})
db.users.find()
exit
```

### Delete pod and verify data survives restart

```bash
# Delete the pod
kubectl delete pod mongodb-0

# Watch it restart with the same name
kubectl get pods

# Exec back in and verify data is still there
kubectl exec -it mongodb-0 -- mongosh
use testDB
db.users.find()
exit

# Verify PVC is still bound
kubectl get pvc
```

### Scale the StatefulSet

```bash
# Scale from 3 to 5 replicas
kubectl scale statefulset mongodb --replicas=5

# Watch sequential creation of pods 3 and 4
kubectl get pods -w
kubectl get pods

# Verify all PVCs bound
kubectl get pvc
```

### Verify data on worker node

```bash
ssh node01
cd /mnt/data
find .
exit
```

### Cleanup

```bash
kubectl delete statefulset mongodb
kubectl delete svc db-service
kubectl delete pvc --all
kubectl delete pv --all
kubectl delete sc db-sc
```

## YAML Files

### mongo-svc.yaml — Headless Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: db-service
  labels:
    app: db
spec:
  clusterIP: None
  selector:
    app: db
  ports:
    - port: 27017
      targetPort: 27017
```

### sc.yaml — StorageClass (local static provisioning)

```yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: db-sc
provisioner: kubernetes.io/no-provisioner
volumeBindingMode: WaitForFirstConsumer
reclaimPolicy: Retain
```

### ss.yaml — StatefulSet (MongoDB)

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mongodb
spec:
  serviceName: db-service
  replicas: 3
  selector:
    matchLabels:
      app: db
  template:
    metadata:
      labels:
        app: db
    spec:
      containers:
        - name: mongodb
          image: mongo:4.4
          ports:
            - containerPort: 27017
          volumeMounts:
            - name: mongo-data
              mountPath: /data/db
  volumeClaimTemplates:
    - metadata:
        name: mongo-data
      spec:
        accessModes:
          - ReadWriteOnce
        storageClassName: db-sc
        resources:
          requests:
            storage: 1Gi
```
