# Dockerizing an Application - Hands-On Demo

📺 [Watch on YouTube](https://www.youtube.com/watch?v=nfRsPiRGx74)

## Overview

Video 2 in the CKA series. Hands-on demo of dockerizing a sample to-do list application. Covers writing a Dockerfile from scratch, building a Docker image, pushing to Docker Hub, pulling from registry, running a container, port binding, accessing the application, and basic container troubleshooting commands (exec, ps, images).

```bash
mkdir day02-code

ls

ls

ls -lrt
```

## Dockerfile

```bash
touch Dockerfile
vi Dockerfile
```

Press `i` to enter insert mode, then write the Dockerfile:

```yaml
FROM node:18-alpine
WORKDIR /app
COPY . .
RUN yarn install --production
CMD ["node", "src/index.js"]
EXPOSE 3000
```

Save and exit: `Esc` → `:wq!`

## Terminal Session: Build the Docker Image

```bash
ls
docker --help
docker build --help
docker build -t day02-todo .
```

## Terminal Session: Verify the Image

```bash
docker images
```

## Terminal Session: Tag and Push to Docker Hub

```bash
# Tag the image for Docker Hub
docker tag day02-todo <username>/test-repo:latest

# Verify the new tag
docker images

# Login to Docker Hub
docker login

# Push the image
docker push <username>/test-repo:latest
```

## Terminal Session: Pull and Run the Container

```bash
# Pull the image (from Docker Hub)
docker pull <username>/test-repo:latest

# Run the container in detached mode with port mapping
docker run -d -p 3000:3000 <username>/test-repo:latest

# Verify container is running
docker ps
```

Access the application at: http://localhost:3000

## Terminal Session: Troubleshooting - Exec into Container

```bash
# Get inside the container (interactive mode)
docker exec -it friendly_panini sh

# Inside the container:
ls
exit
```

> **Note:** Alpine images use `sh` not `bash`. Running `bash` will fail with "executable not found".
