# Video 28 of 59 - Setting Up a Multi-Node Kubernetes Cluster with kubeadm

📺 https://www.youtube.com/watch?v=WcdMC3Lj4tU

## Overview

This video walks through setting up a multi-node Kubernetes cluster on cloud VMs using the kubeadm tool. It covers the complete process: provisioning VMs, opening required ports, installing container runtime (containerd), runc, CNI plugins, kubeadm/kubelet/kubectl, initializing the control plane, installing Calico CNI, and joining worker nodes. This is a critical CKA exam topic.

## Required Ports

**Control Plane Node:**
- 6443 - Kubernetes API server
- 2379-2380 - etcd server client API
- 10248-10260 - kubelet, kube-scheduler (10259), controller-manager (10257)
- 22 - SSH access for admin

**Worker Nodes:**
- 10250 - Kubelet API
- 10256 - kube-proxy
- 30000-32767 - NodePort services
- 22 - SSH access for admin

## Terminal Sessions

### SSH into Control Plane Node

```bash
cd downloads
ssh -i "cka2024.pem" ubuntu@<control-plane-public-ip>
```

### Step 1: Disable Swap (All Nodes)

```bash
sudo swapoff -a
sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab
```

### Step 2: Update Kernel Parameters (All Nodes)

```bash
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter

cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system

# Verify the parameters are set to 1
sysctl net.bridge.bridge-nf-call-iptables net.bridge.bridge-nf-call-ip6tables net.ipv4.ip_forward
```

### Step 3: Install containerd (All Nodes)

```bash
# Download containerd binary
wget https://github.com/containerd/containerd/releases/download/v1.7.4/containerd-1.7.4-linux-amd64.tar.gz

# Extract to /usr/local
sudo tar Cxzvf /usr/local containerd-1.7.4-linux-amd64.tar.gz

# Download containerd service file
wget https://raw.githubusercontent.com/containerd/containerd/main/containerd.service

# Set up systemd service
sudo mkdir -p /usr/local/lib/systemd/system
sudo mv containerd.service /usr/local/lib/systemd/system/

# Generate default containerd config
sudo mkdir -p /etc/containerd
sudo containerd config default | sudo tee /etc/containerd/config.toml

# Update SystemdCgroup from false to true
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/g' /etc/containerd/config.toml

# Start and enable containerd
sudo systemctl daemon-reload
sudo systemctl enable --now containerd

# Verify containerd is running
sudo systemctl status containerd
```

### Step 4: Install runc (All Nodes)

```bash
wget https://github.com/opencontainers/runc/releases/download/v1.1.12/runc.amd64
sudo install -m 755 runc.amd64 /usr/local/sbin/runc
```

### Step 5: Install CNI Plugins (All Nodes)

```bash
wget https://github.com/containernetworking/plugins/releases/download/v1.4.0/cni-plugins-linux-amd64-v1.4.0.tgz
sudo mkdir -p /opt/cni/bin
sudo tar Cxzvf /opt/cni/bin cni-plugins-linux-amd64-v1.4.0.tgz
```

### Step 6: Install kubeadm, kubelet, kubectl (All Nodes)

```bash
sudo apt-get update
sudo apt-get install -y apt-transport-https ca-certificates curl gpg

# Add Kubernetes apt repository key
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

# Add Kubernetes apt repository
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Install packages
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl

# Verify versions
kubeadm version
kubelet --version
kubectl version --client
```

### Step 7: Configure crictl (All Nodes)

```bash
sudo crictl config runtime-endpoint unix:///var/run/containerd/containerd.sock
```

### Step 8: Initialize Control Plane (Master Node Only)

```bash
sudo kubeadm init \
  --apiserver-advertise-address=<master-private-ip> \
  --node-name=master \
  --pod-network-cidr=192.168.0.0/16
```

### Step 9: Configure kubectl (Master Node)

```bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

### Step 10: Install Calico CNI (Master Node)

```bash
# Install Calico operator and CRDs
kubectl create -f https://raw.githubusercontent.com/projectcalico/calico/v3.27.0/manifests/tigera-operator.yaml
kubectl create -f https://raw.githubusercontent.com/projectcalico/calico/v3.27.0/manifests/custom-resources.yaml

# Verify Calico pods
kubectl get pods -A --watch
```

### Step 11: Join Worker Nodes (Worker Nodes Only)

```bash
# Run the join command generated by kubeadm init (on each worker node)
sudo kubeadm join <master-ip>:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash>
```

### Generating a New Join Token (If Lost)

```bash
kubeadm token create --print-join-command
```

### Verify Cluster

```bash
kubectl get nodes
kubectl get pods -A
```

### Reset Control Plane (If Needed)

```bash
sudo kubeadm reset
# Then re-run kubeadm init with corrected parameters
```

### Copy kubeconfig to Worker Nodes (Optional)

```bash
# On worker node, create .kube directory
sudo mkdir -p $HOME/.kube

# Copy admin.conf from master to worker (or copy content manually)
sudo cp /etc/kubernetes/kubelet.conf $HOME/.kube/config
# OR copy admin.conf content from master node into $HOME/.kube/config

sudo chmod 755 $HOME/.kube/config

# Verify
kubectl get nodes
```

### Verify crictl on Worker Nodes

```bash
sudo crictl ps
sudo crictl ps -a
```

### Troubleshooting Permissions

```bash
sudo chmod 775 /var/run/containerd
sudo chmod -R 775 /var/run/containerd/
```

### Setting Terminal PS1 Prompt (For Clarity)

```bash
PS1="master-node$ "
PS1="worker-01$ "
PS1="worker-02$ "
```
