# Video 11 of 59 - Kubernetes Namespaces

https://www.youtube.com/watch?v=yVLXIydlU_0

## Overview

This video covers Kubernetes namespaces — what they are, why they're needed, and how to use them. It demonstrates creating namespaces, deploying resources into specific namespaces, and testing connectivity between pods/services across different namespaces using FQDNs (Fully Qualified Domain Names).

## Key Concepts

- Namespaces provide logical isolation within a cluster
- Default namespaces created by Kubernetes: `default`, `kube-system`, `kube-public`, `kube-node-lease`
- Pods in same namespace can communicate via hostname
- Pods across namespaces must use FQDN: `<service-name>.<namespace>.svc.cluster.local`

## Terminal Sessions

### Viewing existing namespaces and resources

```bash
kubectl get namespaces
# or using alias
k get ns

# Get all resources in kube-system namespace
kubectl get all --namespace kube-system
# or shorter form
kubectl get all -n kube-system

# Get resources in other namespaces
kubectl get all -n kube-public
kubectl get all -n kube-node-lease
kubectl get all -n local-path-storage

# Get resources in default namespace (both equivalent)
kubectl get all -n default
kubectl get all
```

### Creating a namespace (declarative)

```bash
vi ns.yaml
```

```yaml
apiVersion: V1
kind: Namespace
metadata:
  name: demo
```

```bash
kubectl apply -f ns.yaml
kubectl get namespaces
# or
k get ns
```

### Deleting and recreating namespace (imperative)

```bash
kubectl delete ns demo

# Imperative way - faster
kubectl create ns demo
```

### Creating deployments in specific namespaces

```bash
# Create deployment in demo namespace
kubectl create deploy nginx-demo --image=nginx --namespace demo
# or
kubectl create deploy nginx-demo --image=nginx -n demo

# Verify - must specify namespace
kubectl get deploy -n demo

# Create deployment in default namespace
kubectl create deploy nginx-test --image=nginx
kubectl get deploy
```

### Scaling deployments

```bash
# Scale in demo namespace
kubectl scale --replicas=3 deploy nginx-demo --namespace demo

# Scale in default namespace
kubectl scale --replicas=3 deploy nginx-test
```

### Verifying pods with IPs

```bash
# Get pods with IPs in demo namespace
k get pods -o wide -n demo

# Get pods with IPs in default namespace
k get pods -o wide
```

### Exec into pods for connectivity testing

```bash
# Exec into pod in demo namespace
kubectl exec -it <pod-name> -n demo -- sh

# Exec into pod in default namespace
kubectl exec -it <pod-name> -- sh
```

### Testing cross-namespace connectivity via IP

```bash
# From inside a pod, curl another pod's IP (works across namespaces)
curl 10.244.1.7
curl 10.244.2.7
```

### Exposing services

```bash
# Expose deployment in demo namespace
kubectl expose deployment nginx-demo --name svc-demo --port 80 -n demo

# Expose deployment in default namespace
kubectl expose deployment nginx-test --name svc-test --port 80

# Verify services
k get svc -n demo
k get svc
```

### Testing cross-namespace service access

```bash
# Exec into pod
kubectl exec -it <pod-name> -n demo -- sh

# This FAILS (hostname only works within same namespace)
curl svc-test

# Check DNS resolution config
cat /etc/resolv.conf

# This WORKS (using FQDN)
curl svc-test.default.svc.cluster.local

# From default namespace pod to demo namespace service
curl svc-demo.demo.svc.cluster.local
```
